# 第五章：泛型基础与实战

> 本章目标：理解泛型的核心概念——"类型的函数"，并能读懂 Claude Code 中最常见的泛型用法。

---

## 5.1 为什么要用泛型？

假设你需要一个函数，返回数组的最后一个元素：

```ts
// 不用泛型：只能处理 number[]
function lastNumber(arr: number[]): number | undefined {
  return arr[arr.length - 1];
}

// 不用泛型：只能处理 string[]
function lastString(arr: string[]): string | undefined {
  return arr[arr.length - 1];
}
```

同样的逻辑写了两遍！**泛型**可以解决这个问题：

```ts
// 用泛型：一个函数处理所有数组类型
function last<T>(arr: T[]): T | undefined {
  return arr[arr.length - 1];
}

last<number>([1, 2, 3]);      // 返回 number | undefined
last<string>(["a", "b"]);     // 返回 string | undefined
```

`<T>` 叫做**类型参数**，就像函数的参数一样，只不过它接收的是"类型"而不是"值"。

---

## 5.2 Claude Code 中的泛型函数

### 例子 1：array.ts

```ts
// 来自 src/utils/array.ts
export function intersperse<A>(as: A[], separator: (index: number) => A): A[] {
  return as.flatMap((a, i) => (i ? [separator(i), a] : [a]))
}

export function count<T>(arr: readonly T[], pred: (x: T) => unknown): number {
  let n = 0
  for (const x of arr) n += +!!pred(x)
  return n
}

export function uniq<T>(xs: Iterable<T>): T[] {
  return [...new Set(xs)]
}
```

拆解 `intersperse<A>`：
- `<A>` → 声明一个类型参数 `A`
- `as: A[]` → 参数 `as` 是 `A` 类型的数组
- `separator: (index: number) => A` → 参数 `separator` 是一个函数，返回 `A` 类型
- `: A[]` → 函数返回 `A` 类型的数组

TypeScript 通常能**自动推断**泛型参数，所以你调用时不需要写 `<A>`：

```ts
const result = intersperse(["a", "b", "c"], i => "-");
// TypeScript 自动推断 A = string
// result 的类型是 string[]
```

### 例子 2：带约束的泛型

```ts
// 来自 src/types/permissions.ts
export type PermissionAllowDecision<
  Input extends { [key: string]: unknown } = { [key: string]: unknown },
> = {
  behavior: 'allow'
  updatedInput?: Input
}
```

这里 `<Input extends { [key: string]: unknown }>` 表示：

> "`Input` 是一个泛型参数，但它必须是 `{ [key: string]: unknown }` 的子类型。"

`{ [key: string]: unknown }` 是"一个对象，键是字符串，值是 unknown"，也就是 TypeScript 中最宽泛的"对象"类型。（`[key: string]` 这种写法叫做**索引签名**，第九章会详细讲。）

` = { [key: string]: unknown }` 是**默认值**，如果不传泛型参数，就用这个默认类型。

## 5.3 泛型与接口/类型别名

泛型不仅可以用于函数，还可以用于类型：

```ts
// 来自 src/types/permissions.ts
export type PermissionDecision<
  Input extends { [key: string]: unknown } = { [key: string]: unknown },
> =
  | PermissionAllowDecision<Input>
  | PermissionAskDecision<Input>
  | PermissionDenyDecision
```

这里 `PermissionDecision` 本身接收一个 `Input` 泛型参数，然后把它传给内部的 `PermissionAllowDecision<Input>` 和 `PermissionAskDecision<Input>`。

使用时：

```ts
// 不传泛型参数，使用默认值
const decision: PermissionDecision = { behavior: 'allow' };

// 传入具体的 Input 类型
interface MyInput {
  command: string;
  args: string[];
}
const decision2: PermissionDecision<MyInput> = {
  behavior: 'allow',
  updatedInput: { command: 'ls', args: ['-la'] }
};
```

## 5.4 泛型与类型别名：映射类型入门

看一个稍微复杂的例子，来自 `src/types/permissions.ts`：

```ts
export type ToolPermissionRulesBySource = {
  [T in PermissionRuleSource]?: string[]
}
```

这是**映射类型（Mapped Type）**——第七章会详细讲。这里先简单理解：

> "对于 `PermissionRuleSource` 联合类型中的每一个字面量 `T`，都创建一个可选的属性，类型是 `string[]`。"

假设：

```ts
type PermissionRuleSource = 'userSettings' | 'projectSettings' | 'cliArg';
```

那么 `ToolPermissionRulesBySource` 等价于：

```ts
type ToolPermissionRulesBySource = {
  userSettings?: string[];
  projectSettings?: string[];
  cliArg?: string[];
}
```

这展示了泛型的强大：**用类型来生成类型**。

---

## 5.5 泛型的命名约定

在 Claude Code 中，你会看到不同的泛型参数命名：

| 命名 | 常见用途 |
|------|---------|
| `T` | Type（任意类型） |
| `K` | Key（键） |
| `V` | Value（值） |
| `A` | Array element（数组元素） |
| `Input` | 输入类型 |
| `R` | Return type（返回类型） |

```ts
// K 表示 Key，V 表示 Value
function getValue<K extends string, V>(obj: Record<K, V>, key: K): V {
  return obj[key];
}
```

## 5.6 本章小结

| 概念 | 含义 | 例子 |
|------|------|------|
| `<T>` | 类型参数 | `function last<T>(arr: T[]): T` |
| `extends` | 泛型约束 | `<T extends string>` |
| `= Default` | 泛型默认值 | `<T extends object = object>` |
| 类型推断 | 自动推导泛型参数 | `last([1, 2, 3])` → `T = number` |
| 泛型类型 | 类型别名/接口也可以泛型 | `type Box<T> = { value: T }` |

**学习建议**：
- 初学者最容易犯的错误是试图"理解所有泛型"。其实不需要！
- 先做到：**看到 `<T>` 不害怕，能说出"这是一个泛型参数"**
- 再做到：**能根据上下文推断出 `T` 是什么类型**
- 最后才是：**自己会写泛型**
