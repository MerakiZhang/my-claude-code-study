# 第七章：高级类型工具与内置工具类型

> 本章目标：掌握 Claude Code 中频繁出现的高级类型操作，包括 `keyof`、`typeof`、`as const`、映射类型，以及 `Record`、`Partial` 等内置工具类型。

---

## 7.1 keyof：获取对象的所有键

`keyof` 操作符接收一个对象类型，返回它的所有键组成的联合类型。

```ts
interface Theme {
  rainbow_red: string;
  rainbow_orange: string;
  rainbow_blue: string;
}

type ThemeKeys = keyof Theme;
// 等价于：type ThemeKeys = 'rainbow_red' | 'rainbow_orange' | 'rainbow_blue'
```

在 Claude Code 中，`keyof` 最常见的用法是限制参数只能传入合法的键名：

```ts
// 来自 src/utils/thinking.ts
const RAINBOW_COLORS: Array<keyof Theme> = [
  'rainbow_red',
  'rainbow_orange',
  'rainbow_yellow',
  'rainbow_green',
  'rainbow_blue',
  'rainbow_indigo',
  'rainbow_violet',
]

export function getRainbowColor(
  charIndex: number,
  shimmer: boolean = false,
): keyof Theme {
  const colors = shimmer ? RAINBOW_SHIMMER_COLORS : RAINBOW_COLORS
  return colors[charIndex % colors.length]!
}
```

`keyof Theme` 的意思是："必须是 Theme 类型中存在的某个键名"。如果你写错成 `'rainbow_pink'`，TypeScript 会报错。

## 7.2 typeof：提取值的类型

`typeof` 在 TypeScript 中有两种用法：

### 用法 1：运行时 typeof（JavaScript 原生）

```ts
const value = "hello";
console.log(typeof value); // "string"
```

### 用法 2：类型层面的 typeof（TypeScript 独有）

在类型位置上使用 `typeof`，可以**提取一个变量的类型**：

```ts
const config = {
  port: 3000,
  host: 'localhost',
  debug: true,
};

type Config = typeof config;
// 等价于：
// type Config = { port: number; host: string; debug: boolean; }
```

在 Claude Code 中，`typeof` 经常和 `as const` 配合使用：

```ts
// 来自 src/types/permissions.ts
export const EXTERNAL_PERMISSION_MODES = [
  'acceptEdits',
  'bypassPermissions',
  'default',
  'dontAsk',
  'plan',
] as const

export type ExternalPermissionMode = (typeof EXTERNAL_PERMISSION_MODES)[number]
```

这里：
1. `EXTERNAL_PERMISSION_MODES` 是一个数组常量
2. `as const` 把它变成**只读字面量类型**
3. `(typeof EXTERNAL_PERMISSION_MODES)[number]` 提取数组元素的类型

最终 `ExternalPermissionMode` 等价于：

```ts
type ExternalPermissionMode = 'acceptEdits' | 'bypassPermissions' | 'default' | 'dontAsk' | 'plan';
```

## 7.3 infer：从复杂类型中提取类型

`infer` 是 TypeScript 中用于**类型推断**的关键字，通常配合条件类型使用。对于初学者，你不需要深入理解它的底层原理，只需要认识它在 **Zod Schema** 中的常见用法。

Claude Code 使用 Zod 库做运行时数据校验。Zod Schema 本身有类型信息，`z.infer` 可以把这个 Schema 转换成 TypeScript 类型：

```ts
// 假设有一个 Zod Schema
const UserSchema = z.object({
  name: z.string(),
  age: z.number(),
});

// z.infer 提取出 TypeScript 类型
type User = z.infer<typeof UserSchema>;
// 等价于：
// type User = { name: string; age: number; }
```

在 Claude Code 的源码中，这种写法无处不在：

```ts
// 来自 src/types/hooks.ts
export type PromptRequest = z.infer<ReturnType<typeof promptRequestSchema>>

// 来自 src/bridge/bridgePointer.ts
export type BridgePointer = z.infer<ReturnType<typeof BridgePointerSchema>>
```

**初学者只需记住**：看到 `z.infer<...>`，它的意思是"**从这个 Zod Schema 中提取对应的 TypeScript 类型**"。

## 7.4 as const：变成最严格的字面量类型

```ts
// 不用 as const
const modes = ['default', 'plan'];
// 类型推断为：string[]（很宽泛）

// 用 as const
const modes = ['default', 'plan'] as const;
// 类型推断为：readonly ['default', 'plan']（很严格）
```

`as const` 告诉 TypeScript："**不要把我的值推断成宽泛的类型，要保留最精确的字面量类型**。

在 Claude Code 中，`as const` 无处不在：

```ts
// 来自 src/types/permissions.ts
export const INTERNAL_PERMISSION_MODES = [
  ...EXTERNAL_PERMISSION_MODES,
  ...(feature('TRANSCRIPT_CLASSIFIER') ? (['auto'] as const) : ([] as const)),
] as const satisfies readonly PermissionMode[]
```

这里用了三个 TS 特性：
1. `as const` → 变成只读字面量元组
2. `satisfies` → 检查类型是否满足约束（不拓宽类型）
3. `readonly PermissionMode[]` → 只读数组类型

## 7.5 映射类型（Mapped Types）

映射类型可以**基于一个已有类型生成新类型**。

### 基础语法

```ts
type Record<K extends keyof any, T> = {
  [P in K]: T;
};
```

这是 TypeScript 内置的 `Record` 类型的定义！拆解：
- `K extends keyof any` → `K` 必须是 `string | number | symbol` 的子类型（即可以作为对象键）
- `[P in K]` → 对于 `K` 中的每一个键 `P`
- `: T` → 属性的类型是 `T`

### Claude Code 中的 Record

```ts
// 来自 src/utils/permissions/PermissionMode.ts
type PermissionModeConfig = {
  title: string
  shortTitle: string
  symbol: string
  color: ModeColorKey
  external: ExternalPermissionMode
}

const PERMISSION_MODE_CONFIG: Partial<
  Record<PermissionMode, PermissionModeConfig>
> = {
  default: { title: 'Default', shortTitle: 'Default', symbol: '', color: 'text', external: 'default' },
  plan: { title: 'Plan Mode', shortTitle: 'Plan', symbol: PAUSE_ICON, color: 'planMode', external: 'plan' },
  // ...
}
```

`Record<PermissionMode, PermissionModeConfig>` 等价于：

```ts
{
  default: PermissionModeConfig;
  plan: PermissionModeConfig;
  acceptEdits: PermissionModeConfig;
  bypassPermissions: PermissionModeConfig;
  dontAsk: PermissionModeConfig;
  auto?: PermissionModeConfig; // 如果有的话
}
```

`Partial<...>` 表示"所有属性都是可选的"——下面会讲。

## 7.6 内置工具类型

TypeScript 提供了很多内置的工具类型，Claude Code 中常见这些：

### Partial<T>：所有属性变为可选

```ts
interface User {
  name: string;
  age: number;
}

type PartialUser = Partial<User>;
// 等价于：
// { name?: string; age?: number; }
```

在 Claude Code 中：

```ts
const PERMISSION_MODE_CONFIG: Partial<Record<PermissionMode, PermissionModeConfig>>
```

表示"每种 PermissionMode 可能有一个配置，也可能没有"。

### Required<T>：所有属性变为必选

与 `Partial` 相反。

### Pick<T, K>：从 T 中挑选部分属性

```ts
type UserPreview = Pick<User, 'name'>;
// 等价于：{ name: string; }
```

### Omit<T, K>：从 T 中去掉部分属性

```ts
type UserWithoutAge = Omit<User, 'age'>;
// 等价于：{ name: string; }
```

### Exclude<T, U>：从 T 中排除 U

```ts
type T = Exclude<'a' | 'b' | 'c', 'a'>;
// 等价于：'b' | 'c'
```

### Extract<T, U>：从 T 中提取 U

```ts
type T = Extract<'a' | 'b' | 'c', 'a' | 'd'>;
// 等价于：'a'
```

### ReturnType<T>：获取函数返回类型

```ts
function greet() { return 'hello'; }
type GreetReturn = ReturnType<typeof greet>;
// 等价于：string
```

在 Claude Code 中非常常见：

```ts
// 来自源码中的典型用法
let scrollDrainTimer: ReturnType<typeof setTimeout> | undefined
```

`ReturnType<typeof setTimeout>` 获取 `setTimeout` 函数的返回类型（即 `number` 或 `NodeJS.Timeout`，取决于运行环境）。

### Awaited<T>：解包 Promise 的类型

```ts
type Result = Awaited<Promise<string>>;
// 等价于：string
```

Claude Code 中常和 `ReturnType` 组合使用：

```ts
let dirents: Awaited<ReturnType<typeof readdir>>
// 等价于：readdir() 返回的 Promise 解包后的类型
```

### Readonly<T>：把所有属性变成只读

注意区分两个概念：

```ts
// 1. readonly 修饰符（用于属性或数组）
interface User {
  readonly name: string;
}
const nums: readonly number[] = [1, 2, 3];

// 2. Readonly<T> 工具类型（把整个对象的所有属性变成 readonly）
type ReadonlyUser = Readonly<User>;
// 等价于：{ readonly name: string; }
```

`Readonly<T>` 是**浅只读**的——只影响第一层属性。

## 7.7 常用泛型容器：Map 和 Set

除了数组，TypeScript 中还有两种常用的泛型容器：

### Map<K, V>

`Map` 是键值对集合，`K` 是键的类型，`V` 是值的类型：

```ts
// 来自 src/types/permissions.ts
export type ToolPermissionContext = {
  readonly additionalWorkingDirectories: ReadonlyMap<string, AdditionalWorkingDirectory>
}
```

`ReadonlyMap<string, AdditionalWorkingDirectory>` 表示：
- 键是 `string`
- 值是 `AdditionalWorkingDirectory`
- 整个 Map 是只读的（不能添加、删除元素）

### Set<T>

`Set` 是不重复值的集合：

```ts
// 来自 src/commands/clear/conversation.ts
preservedAgentIds.add(task.agentId)
```

这里 `preservedAgentIds` 的类型是 `Set<string>`，表示一个字符串集合。

> **初学者提示**：看到 `Map<string, X>` 就理解为"字符串键、X 值的字典"；看到 `Set<X>` 就理解为"X 类型的不重复集合"。

## 7.9 readonly：只读修饰符（复习与区分）

```ts
// 来自 src/utils/array.ts
export function count<T>(arr: readonly T[], pred: (x: T) => unknown): number {
  // ...
}
```

`readonly T[]` 表示"只读数组"——函数承诺不会修改这个数组。

```ts
const nums: readonly number[] = [1, 2, 3];
nums.push(4); // ❌ 报错！readonly 数组不能 push
```

在对象属性前加 `readonly`：

```ts
// 来自 src/types/permissions.ts
export type ToolPermissionContext = {
  readonly mode: PermissionMode
  readonly additionalWorkingDirectories: ReadonlyMap<string, AdditionalWorkingDirectory>
  // ...
}
```

`ReadonlyMap` 是 TypeScript 内置的只读 Map 类型。

## 7.10 satisfies 运算符

TypeScript 4.9 引入了 `satisfies`，Claude Code 大量使用：

```ts
const PERMISSION_RULE_SOURCES = [
  ...SETTING_SOURCES,
  'cliArg',
  'command',
  'session',
] as const satisfies readonly PermissionRuleSource[]
```

`satisfies` 的作用是：
1. **检查**右边的类型约束是否满足
2. **不改变**左边的具体类型（不像 `: Type` 会拓宽类型）

对比：

```ts
// 用 : 标注类型——会拓宽为宽泛的类型
const modes: string[] = ['default', 'plan'] as const;
// modes 的类型是 string[]，失去了具体的字面量信息

// 用 satisfies——保留具体类型
const modes = ['default', 'plan'] as const satisfies readonly string[];
// modes 的类型是 readonly ['default', 'plan']，保留了字面量
```

## 7.11 本章小结

| 工具 | 作用 | 例子 |
|------|------|------|
| `keyof T` | 获取 T 的所有键 | `keyof Theme` |
| `typeof x` | 提取变量 x 的类型 | `typeof EXTERNAL_PERMISSION_MODES` |
| `infer` | 从复杂类型中提取子类型 | `z.infer<typeof Schema>` |
| `as const` | 变成最严格的字面量类型 | `['a'] as const` |
| `satisfies` | 检查类型约束，不拓宽类型 | `x satisfies string[]` |
| `Record<K, V>` | 键值对对象类型 | `Record<string, number>` |
| `Partial<T>` | 所有属性可选 | `Partial<User>` |
| `Required<T>` | 所有属性必选 | `Required<User>` |
| `Pick<T, K>` | 挑选属性 | `Pick<User, 'name'>` |
| `Omit<T, K>` | 排除属性 | `Omit<User, 'age'>` |
| `ReturnType<T>` | 获取函数返回类型 | `ReturnType<typeof fn>` |
| `Awaited<T>` | 解包 Promise 类型 | `Awaited<Promise<string>>` |
| `Readonly<T>` | 所有属性只读 | `Readonly<User>` |
| `Map<K, V>` | 键值对集合 | `Map<string, number>` |
| `Set<T>` | 不重复值集合 | `Set<string>` |
| `readonly` | 只读修饰 | `readonly T[]` |
