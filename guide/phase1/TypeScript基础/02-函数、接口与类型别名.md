# 第二章：函数、接口与类型别名

> 本章目标：掌握 TypeScript 中定义函数的多种方式，理解 `type` 和 `interface` 的区别与选用场景。

---

## 2.1 函数的多种写法

在 Claude Code 中，定义函数主要有三种方式。**作为零基础学生，你需要全部认识它们。**

### 写法一：函数声明（最常见）

```ts
// 来自 src/Task.ts
export function generateTaskId(type: TaskType): string {
  const prefix = getTaskIdPrefix(type)
  const bytes = randomBytes(8)
  let id = prefix
  for (let i = 0; i < 8; i++) {
    id += TASK_ID_ALPHABET[bytes[i]! % TASK_ID_ALPHABET.length]
  }
  return id
}
```

拆解：
- `export function` → 导出这个函数，其他文件可以导入使用
- `generateTaskId` → 函数名
- `(type: TaskType)` → 接收一个参数 `type`，类型为 `TaskType`
- `: string` → 返回类型为 `string`

### 写法二：箭头函数（常用于回调和类型定义）

```ts
// 来自 src/Task.ts
export type SetAppState = (f: (prev: AppState) => AppState) => void
```

这是一个**函数类型**的定义，不是实际的函数。它表示：

> "`SetAppState` 是一种函数类型，它接收一个参数 `f`（`f` 本身也是一个函数），返回值是 `void`。"

如果把它展开写，等价于：

```ts
// 参数 f 的类型
interface StateUpdater {
  (prev: AppState): AppState;
}

// SetAppState 本身
interface SetAppState {
  (f: StateUpdater): void;
}
```

箭头函数类型 `(参数) => 返回值` 在 Claude Code 中极其常见。

### 写法三：函数作为对象的方法

```ts
// 来自 src/Task.ts
export type Task = {
  name: string
  type: TaskType
  kill(taskId: string, setAppState: SetAppState): Promise<void>
}
```

这里 `kill` 是 `Task` 对象的一个方法：
- 接收 `taskId`（`string`）和 `setAppState`（`SetAppState` 类型）
- 返回 `Promise<void>`（异步操作，第六章讲）

## 2.2 可选参数与默认参数

### 可选参数（用 `?` 标记）

```ts
// 来自 src/utils/treeify.ts
export function treeify(obj: TreeNode, options: TreeifyOptions = {}): string {
  // ...
}
```

这里 `options` 参数有默认值 `{}`，所以不传也可以。

再看一个更清晰的例子：

```ts
// 来自 src/utils/thinking.ts
export function getRainbowColor(
  charIndex: number,
  shimmer: boolean = false,  // ← 默认参数
): keyof Theme {
  const colors = shimmer ? RAINBOW_SHIMMER_COLORS : RAINBOW_COLORS
  return colors[charIndex % colors.length]!
}
```

- `shimmer: boolean = false` → 如果不传第二个参数，默认为 `false`
- `keyof Theme` → 第七章会讲，表示"Theme 类型的键名"

### 可选属性 vs 可选参数

```ts
// 对象中的可选属性（用 ?）
export type TreeifyOptions = {
  showValues?: boolean    // 可以不传
  hideFunctions?: boolean // 可以不传
  useColors?: boolean     // 可以不传
  themeName?: ThemeName   // 可以不传
}

// 函数中的可选参数（也用 ?）
function greet(name: string, greeting?: string): string {
  return `${greeting ?? 'Hello'}, ${name}`;
}
```

**规律**：无论在对象还是函数中，`?` 都表示"可以不存在"。

## 2.4 剩余参数（Rest Parameters）

当函数的参数数量不固定时，用 `...` 表示"剩余的所有参数"：

```ts
// 来自 src/ink/termio/osc.ts
export function osc(...parts: (string | number)[]): string {
  // parts 是一个数组，包含所有传入的参数
  return `\x1b]` + parts.join(';') + '\x07'
}

// 调用时可以传任意多个
osc('1337', 'File=name')
osc('1337', 'File=name', 'Size=1024')
```

`...parts: (string | number)[]` 表示：
- `...parts` → 收集所有剩余参数到一个数组
- `(string | number)[]` → 数组元素的类型是 `string` 或 `number`

再看一个来自 `src/state/AppStateStore.ts` 的例子：

```ts
log: (...args: unknown[]) => void
```

这表示 `log` 方法接收任意多个参数，每个参数的类型是 `unknown`（第六章讲过）。

## 2.5 type 与 interface 的区别

这是 TypeScript 中最常被问到的问题之一。Claude Code 源码中**大量使用的是 `type`**，但 `interface` 也有出现。

### 相同点

两者都可以描述对象结构：

```ts
// 用 type
type Person = {
  name: string;
  age: number;
};

// 用 interface
interface Person {
  name: string;
  age: number;
}
```

### 不同点 1：interface 可以「合并声明」

```ts
interface User {
  name: string;
}

interface User {
  age: number;
}

// 最终 User 同时有 name 和 age
const user: User = { name: "Claude", age: 3 }; // ✅
```

`type` 不能这样重复定义同名类型。

### 不同点 2：type 可以定义「联合类型」

这是 Claude Code 大量使用 `type` 的原因：

```ts
// 来自 src/types/permissions.ts
export type PermissionBehavior = 'allow' | 'deny' | 'ask'

export type PermissionRuleSource =
  | 'userSettings'
  | 'projectSettings'
  | 'localSettings'
  | 'flagSettings'
  | 'policySettings'
  | 'cliArg'
  | 'command'
  | 'session'
```

`interface` **不能**定义这种"A 或 B 或 C"的联合类型，只有 `type` 可以。

再看一个更复杂的例子，来自 `src/Tool.ts`：

```ts
export type ValidationResult =
  | { result: true }
  | {
      result: false
      message: string
      errorCode: number
    }
```

这表示 `ValidationResult` 可以是两种对象结构之一：`{ result: true }` 或 `{ result: false, message: string, errorCode: number }`。

### 不同点 3：type 可以用在更多场景

```ts
// type 可以给基础类型起别名
type UserID = string;

// type 可以定义元组
type Point = [number, number];

// type 可以用 typeof 提取类型（第七章讲）
type Config = typeof defaultConfig;
```

`interface` 只能描述对象结构，没有这些能力。

### Claude Code 的选择

在 Claude Code 中：
- **定义对象结构**：`type` 和 `interface` 都有使用
- **定义联合类型**：**只能用 `type`**（这也是 `type` 用得更多的原因）
- **React 组件的 Props**：通常用 `type Props = { ... }`

## 2.6 解构赋值中的类型

在 TypeScript 中，解构对象时也可以标注类型：

```ts
// 来自 src/utils/treeify.ts
export function treeify(obj: TreeNode, options: TreeifyOptions = {}): string {
  const {
    showValues = true,
    hideFunctions = false,
    themeName = 'dark',
    treeCharColors = {},
  } = options
  // ...
}
```

这里 `options` 的类型是 `TreeifyOptions`，解构出来的每个变量都继承了这个类型。

## 2.7 本章小结

| 特性 | `type` | `interface` | Claude Code 使用情况 |
|------|--------|-------------|---------------------|
| 定义对象结构 | ✅ | ✅ | 都用 |
| 定义联合类型 | ✅ | ❌ | **大量使用 `type`** |
| 合并声明 | ❌ | ✅ | 极少用 |
| 给基础类型起别名 | ✅ | ❌ | 偶尔用 |
| React Props | ✅ | 偶尔 | 主要用 `type` |

**对初学者建议**：
1. 如果类型是"A 或 B"的联合类型 → **必须用 `type`**
2. 如果只是描述对象结构 → 用 `type` 或 `interface` 都可以
3. 看团队规范，Claude Code 团队显然偏爱 `type`
