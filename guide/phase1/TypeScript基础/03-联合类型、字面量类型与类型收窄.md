# 第三章：联合类型、字面量类型与类型收窄

> 本章目标：理解 Claude Code 源码中最核心的类型模式——"A 或 B 或 C"的联合类型，以及 TypeScript 如何安全地处理它们。

---

## 3.1 联合类型（Union Types）

联合类型用 `|` 符号表示"**或是这种，或是那种**"：

```ts
type Status = 'pending' | 'running' | 'completed' | 'failed';
type Value = string | number;
```

在 Claude Code 中，**几乎每个核心类型都是联合类型**。这是项目最重要的 TypeScript 模式。

### 例子 1：任务状态

```ts
// 来自 src/Task.ts
export type TaskStatus =
  | 'pending'
  | 'running'
  | 'completed'
  | 'failed'
  | 'killed'
```

一个任务的状态只能是这五种之一，不能是其他字符串。

### 例子 2：Thinking 配置

```ts
// 来自 src/utils/thinking.ts
export type ThinkingConfig =
  | { type: 'adaptive' }
  | { type: 'enabled'; budgetTokens: number }
  | { type: 'disabled' }
```

这里有三种配置模式：
1. `{ type: 'adaptive' }` —— 自适应思考
2. `{ type: 'enabled'; budgetTokens: number }` —— 启用，并指定 token 预算
3. `{ type: 'disabled' }` —— 禁用

注意：第二种模式下**必须**提供 `budgetTokens`，其他两种则不需要。

### 例子 3：权限决策

```ts
// 来自 src/types/permissions.ts
export type PermissionDecision<Input extends { [key: string]: unknown } = { [key: string]: unknown }> =
  | PermissionAllowDecision<Input>
  | PermissionAskDecision<Input>
  | PermissionDenyDecision
```

权限决策只能是"允许"、"询问"或"拒绝"之一。这里用到了泛型（第五章讲），但核心思想是联合类型。

## 3.2 字面量类型（Literal Types）

字面量类型是指**具体的值**作为类型，而不是宽泛的 `string` 或 `number`。

```ts
// 不是任意 string，只能是 'local_bash'
type A = 'local_bash';

// 不是任意 number，只能是 200
type B = 200;

// 不是任意 boolean，只能是 true
type C = true;
```

单独看字面量类型没什么用，但**和联合类型结合**就非常强大：

```ts
// 来自 src/types/permissions.ts
export type PermissionBehavior = 'allow' | 'deny' | 'ask'
```

`PermissionBehavior` 不是任意字符串，只能是 `'allow'`、`'deny'` 或 `'ask'` 这三个**具体的字符串值**。

### 为什么用字面量类型？

**安全性**：如果你拼错了字，TypeScript 会立即报错。

```ts
function setBehavior(b: PermissionBehavior) { /* ... */ }

setBehavior('allow');   // ✅
setBehavior('Allow');   // ❌ 报错！TypeScript 知道只能是全小写的 'allow'
setBehavior('reject');  // ❌ 报错！不在联合类型中
```

**自动补全**：IDE 会提示你可以输入哪些值。

## 3.3  discriminated unions（可辨识联合）

这是联合类型的高级用法，也是 Claude Code 中**最重要的类型模式**。

### 核心思想

联合类型的每个成员都有一个**共同的"标签"属性**（通常是 `type` 或 `kind`），TypeScript 可以根据这个标签来区分不同的成员。

```ts
// 来自 src/types/permissions.ts
export type PermissionUpdate =
  | {
      type: 'addRules'
      destination: PermissionUpdateDestination
      rules: PermissionRuleValue[]
      behavior: PermissionBehavior
    }
  | {
      type: 'replaceRules'
      destination: PermissionUpdateDestination
      rules: PermissionRuleValue[]
      behavior: PermissionBehavior
    }
  | {
      type: 'removeRules'
      destination: PermissionUpdateDestination
      rules: PermissionRuleValue[]
      behavior: PermissionBehavior
    }
  | {
      type: 'setMode'
      destination: PermissionUpdateDestination
      mode: ExternalPermissionMode
    }
  | {
      type: 'addDirectories'
      destination: PermissionUpdateDestination
      directories: string[]
    }
  | {
      type: 'removeDirectories'
      destination: PermissionUpdateDestination
      directories: string[]
    }
```

这里 `type` 就是**可辨识的标签**：
- 如果 `type === 'setMode'`，那对象一定有 `mode` 属性
- 如果 `type === 'addDirectories'`，那对象一定有 `directories` 属性
- 如果 `type === 'addRules'`，那对象一定有 `rules` 和 `behavior` 属性

### 类型收窄（Narrowing）

TypeScript 会根据 `if` 判断自动收窄类型：

```ts
function handleUpdate(update: PermissionUpdate) {
  if (update.type === 'setMode') {
    // 在这个代码块里，TypeScript 知道 update 是 { type: 'setMode', mode: ExternalPermissionMode }
    console.log(update.mode);      // ✅ 可以安全访问 mode
    console.log(update.directories); // ❌ 报错！setMode 类型没有 directories
  } else if (update.type === 'addDirectories') {
    // 在这个代码块里，TypeScript 知道 update 有 directories
    console.log(update.directories); // ✅
    console.log(update.mode);        // ❌ 报错！
  }
}
```

这就是**类型收窄**——TypeScript 像一个聪明的助手，在你做完判断后，自动帮你缩小了类型的范围。

## 3.4 类型守卫（Type Guards）

有时类型收窄需要自定义逻辑，这时用**类型守卫函数**。

### 内置的类型守卫

```ts
// typeof 收窄
function process(value: string | number) {
  if (typeof value === 'string') {
    // value 在这里是 string 类型
    value.toUpperCase(); // ✅
  } else {
    // value 在这里是 number 类型
    value.toFixed(2); // ✅
  }
}
```

在 Claude Code 的 `src/utils/treeify.ts` 中大量使用：

```ts
function growBranch(node: TreeNode | string, prefix: string, _isLast: boolean, depth: number = 0): void {
  if (typeof node === 'string') {
    // node 在这里被收窄为 string
    lines.push(prefix + colorize(node, treeCharColors.value))
    return
  }

  if (typeof node !== 'object' || node === null) {
    // node 在这里被收窄为 null 或 primitive
    if (showValues) {
      const valueStr = String(node)
      lines.push(prefix + colorize(valueStr, treeCharColors.value))
    }
    return
  }
  // 到这里，TypeScript 知道 node 是 TreeNode（对象类型）
}
```

### 自定义类型守卫（`is` 关键字）

```ts
// 来自 src/utils/permissions/PermissionMode.ts
export function isExternalPermissionMode(
  mode: PermissionMode,
): mode is ExternalPermissionMode {
  // External users can't have auto, so always true for them
  if (process.env.USER_TYPE !== 'ant') {
    return true
  }
  return mode !== 'auto' && mode !== 'bubble'
}
```

这个函数的返回类型不是普通的 `boolean`，而是 `mode is ExternalPermissionMode`。这告诉 TypeScript：

> "如果函数返回 `true`，那么 `mode` 就是 `ExternalPermissionMode` 类型。"

使用时：

```ts
const mode: PermissionMode = 'default';

if (isExternalPermissionMode(mode)) {
  // 在这个代码块里，mode 被收窄为 ExternalPermissionMode
  toExternalPermissionMode(mode); // ✅
}
```

再看一个来自 `src/tasks/types.ts` 的例子（假设）：

```ts
export function isBackgroundTask(task: Task): task is BackgroundTask {
  return task.type === 'background';
}
```

## 3.5 `in` 操作符收窄

```ts
function handleResult(result: { data: string } | { error: string }) {
  if ('data' in result) {
    // result 在这里是 { data: string }
    console.log(result.data);
  } else {
    // result 在这里是 { error: string }
    console.log(result.error);
  }
}
```

## 3.6 本章小结

| 概念 | 符号/写法 | 作用 |
|------|----------|------|
| 联合类型 | `A \| B` | "或是A，或是B" |
| 字面量类型 | `'pending'`、`200`、`true` | 限定为具体值 |
| 可辨识联合 | `{ type: 'A', ... } \| { type: 'B', ... }` | 用共同标签区分不同形状 |
| 类型收窄 | `if`、`switch`、`typeof` | 缩小变量类型范围 |
| 类型守卫 | `x is Type` | 自定义类型判断逻辑 |

**Claude Code 的核心设计模式**：几乎所有业务概念都用"可辨识联合"建模——任务状态、权限决策、消息类型、工具结果……这是阅读源码最重要的类型知识。
