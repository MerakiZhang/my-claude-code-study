# 第五章：数组与对象的高级操作

> 本章目标：学会用简洁优雅的方式处理数组和对象，而不是写一堆 for 循环。
>
> 想象你有一份购物清单。传统的 `for` 循环就像你亲自一个一个检查商品。而高阶函数就像请了一个助手——你说"给我找出所有水果"，助手直接帮你筛选好，你不需要自己动手。

---

## 5.1 高阶函数概念

**高阶函数**（Higher-Order Function）听起来很高深，其实概念很简单：

> **高阶函数 = 接收函数作为参数，或者返回一个函数的函数。**

```js
// 普通函数：接收数字，返回数字
function double(x) {
  return x * 2;
}

// 高阶函数：接收函数，返回新函数
function createMultiplier(factor) {
  return function(x) {
    return x * factor;
  };
}

const triple = createMultiplier(3);
console.log(triple(5)); // 15
```

> 💡 **类比**：高阶函数就像定制工厂——你不是买现成的产品，而是给一个"规格说明"（函数），工厂按你的规格生产。

**回调函数**（Callback）是指作为参数传给另一个函数的函数：

```js
function doSomething(callback) {
  console.log('做点什么...');
  callback();  // 执行传入的函数
}

doSomething(function() {
  console.log('做完了！');
});

// 输出：
// 做点什么...
// 做完了！
```

JavaScript 数组的内置方法（如 `map`、`filter`）都是高阶函数——它们接收一个回调函数，然后用这个回调处理数组的每个元素。

> 💡 **与 TypeScript 泛型的联系**：TypeScript 中的泛型（如 `function last<T>(arr: T[])`）描述的是"这个函数可以处理任意类型的数组"。但在 JavaScript 运行时，我们不需要写 `<T>`——因为 JS 是**动态类型语言**，`map`、`filter`、`reduce` 这些高阶函数天然就能处理字符串数组、数字数组、对象数组，根本不需要事先声明"我要处理什么类型"。TypeScript 的泛型只是在编译时给这种"天然泛型"的能力加上类型约束，编译成 JavaScript 后所有的 `<T>` 都会被完全擦除。

## 5.2 数组的高阶方法

### forEach —— 遍历每个元素

```js
const fruits = ['苹果', '香蕉', '橙子'];

fruits.forEach(function(fruit, index) {
  console.log(index + ': ' + fruit);
});

// 输出：
// 0: 苹果
// 1: 香蕉
// 2: 橙子
```

`forEach` 就像点名——依次对每个学生（元素）做同样的事。注意：**它不返回任何值**，只是执行副作用（比如打印、修改外部变量）。

> 💡 **类比**：`forEach` 就像超市收银员——一个一个扫描商品，不给你新清单，只是记账。

---

### map —— 映射（一一对应地转换）

```js
const numbers = [1, 2, 3, 4];

const doubled = numbers.map(function(n) {
  return n * 2;
});

console.log(doubled); // [2, 4, 6, 8]
console.log(numbers); // [1, 2, 3, 4]（原数组不变！）
```

`map` 会创建一个新数组，原数组的每个元素经过回调函数处理后放入新数组。**原数组不会改变**。

> 💡 **类比**：`map` 就像翻译官——你说一句话，他翻译成外语。你说三句话，他给你三句翻译。不改变你说的原话。

用箭头函数写更简洁：

```js
const doubled = numbers.map(n => n * 2);
```

---

### filter —— 过滤（筛选符合条件的）

```js
const numbers = [1, 2, 3, 4, 5, 6];

const evens = numbers.filter(function(n) {
  return n % 2 === 0;  // 只保留能被 2 整除的
});

console.log(evens); // [2, 4, 6]
```

回调函数返回 `true` 的元素会被保留，返回 `false` 的会被过滤掉。

> 💡 **类比**：`filter` 就像筛子——把细沙漏下去，大石头留在上面。

---

### find —— 查找第一个匹配的

```js
const users = [
  { id: 1, name: '小明' },
  { id: 2, name: '小红' },
  { id: 3, name: '小刚' },
];

const user = users.find(function(u) {
  return u.id === 2;
});

console.log(user); // { id: 2, name: '小红' }
```

找到**第一个**满足条件的元素就停下来，返回该元素。如果找不到，返回 `undefined`。

> 💡 **类比**：`find` 就像找钥匙——在抽屉里翻，找到第一把对的钥匙就停下，不继续翻了。

---

### findIndex —— 查找第一个匹配的索引

和 `find` 类似，但返回的是索引而不是元素本身：

```js
const fruits = ['苹果', '香蕉', '橙子'];

const index = fruits.findIndex(function(fruit) {
  return fruit === '橙子';
});

console.log(index); // 2
```

找不到时返回 `-1`。

---

### some —— 是否有至少一个满足条件

```js
const scores = [65, 72, 88, 59, 91];

const hasPassing = scores.some(function(score) {
  return score >= 60;
});

console.log(hasPassing); // true（至少有一个人及格了）
```

只要有一个元素满足条件就返回 `true`，否则返回 `false`。

> 💡 **类比**：`some` 就像问"班里有人去过北京吗？"——只要有一个人举手，答案就是"有"。

---

### every —— 是否全部满足条件

```js
const scores = [85, 92, 78, 88, 95];

const allPassing = scores.every(function(score) {
  return score >= 60;
});

console.log(allPassing); // true（所有人都及格了）
```

所有元素都满足条件才返回 `true`，否则返回 `false`。

> 💡 **类比**：`every` 就像查作业——所有人都交了，才算"齐"。有一个人没交，就是"不齐"。

---

### reduce —— 归约/累加

`reduce` 是最强大的数组方法，但也最容易让人困惑。它的作用是把数组"归约"成一个值。

```js
const numbers = [1, 2, 3, 4, 5];

const sum = numbers.reduce(function(acc, current) {
  return acc + current;
}, 0);  // 0 是初始值

console.log(sum); // 15
```

**工作原理**：

| 步骤 | acc（累加器） | current（当前值） | 返回值 |
|------|-------------|-----------------|--------|
| 1 | 0（初始值） | 1 | 0 + 1 = 1 |
| 2 | 1 | 2 | 1 + 2 = 3 |
| 3 | 3 | 3 | 3 + 3 = 6 |
| 4 | 6 | 4 | 6 + 4 = 10 |
| 5 | 10 | 5 | 10 + 5 = 15 |

> 💡 **类比**：`reduce` 就像算账——你手里有个计算器（acc），一个一个加数字（current），最后得到总数。

不只是求和，还可以做很多其他事情：

```js
// 找出最大值
const max = [3, 1, 4, 1, 5].reduce(function(acc, n) {
  return n > acc ? n : acc;
}, -Infinity);  // 提供初始值，避免空数组时报错

// 把数组转成对象（按 id 索引）
const users = [
  { id: 1, name: '小明' },
  { id: 2, name: '小红' },
];

const userMap = users.reduce(function(acc, user) {
  acc[user.id] = user;
  return acc;
}, {});

console.log(userMap);
// { '1': { id: 1, name: '小明' }, '2': { id: 2, name: '小红' } }
```

---

### flatMap —— 先映射，再扁平化

`flatMap` = `map` + `flat(1)`。先对每个元素做映射，然后把结果"拍平"一层。

```js
const sentences = ['Hello world', 'Good morning'];

const words = sentences.flatMap(function(sentence) {
  return sentence.split(' ');
});

console.log(words); // ['Hello', 'world', 'Good', 'morning']
```

如果用 `map`，会得到嵌套数组：

```js
const mapped = sentences.map(s => s.split(' '));
// [['Hello', 'world'], ['Good', 'morning']]

// flatMap 帮你自动拍平一层
```

> 💡 **类比**：`flatMap` 就像把两层抽屉合并成一层——先取出每个小抽屉里的东西，再把它们放到一个大抽屉里。

---

### includes —— 是否包含某元素

```js
const fruits = ['苹果', '香蕉', '橙子'];

console.log(fruits.includes('香蕉')); // true
console.log(fruits.includes('葡萄')); // false
```

简单直接：数组里有这个值吗？

---

### indexOf / lastIndexOf —— 查找索引

```js
const numbers = [10, 20, 30, 20, 40];

console.log(numbers.indexOf(20));     // 1（第一个 20 的位置）
console.log(numbers.lastIndexOf(20)); // 3（最后一个 20 的位置）
console.log(numbers.indexOf(99));     // -1（找不到）
```

和 `findIndex` 的区别：`indexOf` 是直接比较值是否相等，`findIndex` 是用回调函数判断条件。

### 源码例子：Claude Code 的数组工具函数

在 Claude Code 的 `src/utils/array.js` 中（转换为纯 JavaScript），有几个很实用的工具函数：

```js
// 来自 src/utils/array.js（转换为纯 JavaScript）

// intersperse：在数组元素之间插入分隔符
// 比如 intersperse(['a', 'b', 'c'], i => '-') 得到 ['a', '-', 'b', '-', 'c']
export function intersperse(as, separator) {
  return as.flatMap(function(a, i) {
    return i ? [separator(i), a] : [a];
  });
}

// 拆解：
// 1. flatMap 遍历数组，同时有 map + flat 的效果
// 2. i 是索引，从 0 开始
// 3. i ? [separator(i), a] : [a]
//    - 当 i 是 0（第一个元素），只返回 [a]
//    - 当 i 不是 0，返回 [分隔符, a]
// 4. flatMap 把所有结果拍平成一个数组

console.log(intersperse(['苹果', '香蕉', '橙子'], i => '→'));
// ['苹果', '→', '香蕉', '→', '橙子']


// count：统计数组中满足条件的元素个数
export function count(arr, pred) {
  let n = 0;
  for (const x of arr) {
    if (pred(x)) {
      n++;
    }
  }
  return n;
}

// 拆解：
// 1. pred 是一个函数，接收数组元素，返回真假值
// 2. 如果 pred(x) 的结果是真值（如 true、1、非空字符串等），n 加 1
// 3. 遍历完整个数组，返回总数

const scores = [55, 82, 91, 60, 43];
const passingCount = count(scores, s => s >= 60);
console.log(passingCount); // 3


// uniq：数组去重
export function uniq(xs) {
  return [...new Set(xs)];
}

// 拆解：
// 1. new Set(xs) 把数组转成 Set（集合会自动去重）
// 2. [...set] 用展开运算符把 Set 转回数组

console.log(uniq([1, 2, 2, 3, 3, 3])); // [1, 2, 3]
```

## 5.3 数组去重

`Set` 是 ES6 引入的一种数据结构，它的特点是**成员不能重复**。

```js
const set = new Set([1, 2, 2, 3, 3, 3]);
console.log(set); // Set { 1, 2, 3 }

// 转回数组
const unique = [...set];
console.log(unique); // [1, 2, 3]

// 一行代码去重
const arr = [1, 2, 2, 3, 3, 3];
const uniqueArr = [...new Set(arr)];
```

> 💡 **类比**：`Set` 就像班里的花名册——同一个名字只登记一次，不会重复。

`Set` 也支持 `add`、`has`、`delete` 等操作：

```js
const s = new Set();
s.add('苹果');
s.add('香蕉');
s.add('苹果');  // 重复添加无效

console.log(s.has('苹果')); // true
console.log(s.size);        // 2
```

## 5.4 Object 的静态方法

JavaScript 提供了一些方便的工具方法来操作对象。

### Object.keys —— 获取所有键

```js
const person = { name: '小明', age: 20, city: '北京' };

const keys = Object.keys(person);
console.log(keys); // ['name', 'age', 'city']
```

返回一个数组，包含对象所有"自己的、可枚举的"属性名。

---

### Object.values —— 获取所有值

```js
const person = { name: '小明', age: 20, city: '北京' };

const values = Object.values(person);
console.log(values); // ['小明', 20, '北京']
```

在 Claude Code 的源码中很常见：

```js
// 来自 Claude Code 源码的概念转换
const tasks = {
  task1: { status: 'running' },
  task2: { status: 'completed' },
  task3: { status: 'pending' },
};

// 获取所有任务
const allTasks = Object.values(tasks);
console.log(allTasks);
// [{ status: 'running' }, { status: 'completed' }, { status: 'pending' }]

// 统计运行中的任务数
const runningCount = allTasks.filter(t => t.status === 'running').length;
console.log(runningCount); // 1
```

---

### Object.entries —— 获取键值对

```js
const person = { name: '小明', age: 20 };

const entries = Object.entries(person);
console.log(entries);
// [['name', '小明'], ['age', 20]]
```

返回一个二维数组，每个元素是一个 `[key, value]` 数组。

配合 `forEach` 或 `for...of` 使用：

```js
const scores = { 小明: 85, 小红: 92, 小刚: 78 };

Object.entries(scores).forEach(function([name, score]) {
  console.log(name + '考了' + score + '分');
});

// 输出：
// 小明考了85分
// 小红考了92分
// 小刚考了78分
```

---

### Object.assign —— 合并对象

```js
const defaults = { theme: 'dark', fontSize: 14 };
const userSettings = { fontSize: 16 };

const settings = Object.assign({}, defaults, userSettings);
console.log(settings);
// { theme: 'dark', fontSize: 16 }
```

`Object.assign(target, source1, source2, ...)` 把后面所有对象的属性复制到第一个对象里。**后面的属性会覆盖前面的同名属性**。

> ⚠️ 注意：`Object.assign` 是浅拷贝。如果属性值是对象，复制的是引用而不是新对象。

现代 JavaScript 更推荐使用展开运算符：

```js
const settings = { ...defaults, ...userSettings };
// 效果相同，写法更简洁
```

---

### Object.hasOwn —— 检查自有属性

```js
const obj = { name: '小明' };

console.log(Object.hasOwn(obj, 'name'));   // true
console.log(Object.hasOwn(obj, 'toString')); // false

// 注意：obj.toString() 是可以调用的，但 toString 不是 obj 自己的属性
// 而是从原型链继承来的
```

`Object.hasOwn` 和 `in` 运算符的区别：
- `Object.hasOwn` 只检查对象**自己**的属性
- `'prop' in obj` 还会检查原型链上继承的属性

---

### Object.fromEntries —— 从键值对数组创建对象

`Object.entries` 的逆操作：

```js
const entries = [['name', '小明'], ['age', 20]];

const person = Object.fromEntries(entries);
console.log(person); // { name: '小明', age: 20 }
```

配合 `map` 和 `filter` 使用非常强大：

```js
const scores = { 小明: 55, 小红: 82, 小刚: 91 };

// 过滤出及格的学生，再转回对象
const passed = Object.fromEntries(
  Object.entries(scores).filter(function([name, score]) {
    return score >= 60;
  })
);

console.log(passed); // { 小红: 82, 小刚: 91 }
```

> 💡 **类比**：`Object.entries` 和 `Object.fromEntries` 就像一对翻译官——一个把对象翻译成数组，一个把数组翻译回对象。中间你可以用数组的方法做各种操作。

## 5.5 数组和字符串的互相转换

### split —— 字符串拆成数组

```js
const str = 'Hello World';

console.log(str.split(''));      // ['H', 'e', 'l', 'l', 'o', ' ', 'W', 'o', 'r', 'l', 'd']
console.log(str.split(' '));     // ['Hello', 'World']
console.log(str.split('o'));     // ['Hell', ' W', 'rld']
```

### join —— 数组拼成字符串

```js
const words = ['Hello', 'World'];

console.log(words.join(' '));   // 'Hello World'
console.log(words.join('-'));   // 'Hello-World'
console.log(words.join(''));    // 'HelloWorld'
console.log(words.join());      // 'Hello,World'（默认用逗号）
```

> 💡 **类比**：`split` 就像切蛋糕——一刀切下去，分成几块。`join` 就像拼拼图——把几块拼成一整幅。

### 实用组合

```js
// 反转字符串
const reverse = str => str.split('').reverse().join('');
console.log(reverse('hello')); // 'olleh'

// 统计单词数
const wordCount = str => str.trim().split(/\s+/).filter(w => w).length;
console.log(wordCount('Hello   World')); // 2
```

## 本章小结

| 方法 | 作用 | 是否改变原数组 | 返回值 |
|------|------|-------------|--------|
| `forEach` | 遍历每个元素 | 否 | `undefined` |
| `map` | 映射转换 | 否 | 新数组 |
| `filter` | 过滤元素 | 否 | 新数组 |
| `find` | 找第一个匹配项 | 否 | 元素或 `undefined` |
| `findIndex` | 找第一个匹配的索引 | 否 | 索引或 `-1` |
| `some` | 是否有满足的 | 否 | `boolean` |
| `every` | 是否全部满足 | 否 | `boolean` |
| `reduce` | 归约成一个值 | 否 | 任意值 |
| `flatMap` | 映射后扁平化 | 否 | 新数组 |
| `includes` | 是否包含 | 否 | `boolean` |
| `indexOf` | 查找值的索引 | 否 | 索引或 `-1` |

| Object 方法 | 作用 |
|-------------|------|
| `Object.keys(obj)` | 获取所有键（数组） |
| `Object.values(obj)` | 获取所有值（数组） |
| `Object.entries(obj)` | 获取键值对（二维数组） |
| `Object.assign(target, source)` | 合并对象 |
| `Object.hasOwn(obj, key)` | 检查自有属性 |
| `Object.fromEntries(arr)` | 从键值对数组创建对象 |

| 其他 | 作用 |
|------|------|
| `new Set(arr)` | 创建去重集合 |
| `[...set]` | Set 转数组 |
| `str.split(sep)` | 字符串拆数组 |
| `arr.join(sep)` | 数组拼字符串 |

**核心原则**：
1. 优先用高阶方法，少写 `for` 循环——代码更清晰、意图更明确
2. `map`、`filter` 不会改变原数组，总是返回新数组
3. `Object.entries` + 数组方法 + `Object.fromEntries` 是处理对象的黄金组合
4. `[...new Set(arr)]` 是一行去重的标准写法
