# 第二章：函数与对象

> 本章目标：掌握 JavaScript 中函数的多种写法，理解参数的高级用法，学会解构赋值和展开运算符。

---

## 2.1 函数的三种写法

### 什么是函数？

函数就像一台**自动售货机**：你投入硬币（传入参数），按下按钮（调用函数），机器内部处理一番，最后吐出饮料（返回结果）。

函数的作用是把一段"经常要用的代码"打包起来，给它起个名字，以后需要时直接调用。

### 写法一：函数声明（最常见）

```js
function add(a, b) {
  return a + b;
}

console.log(add(3, 5));  // 输出：8
```

拆解：
- `function` → 声明"我要定义一个函数"
- `add` → 函数的名字
- `(a, b)` → 参数列表，表示这台机器接收两个输入
- `{ ... }` → 函数体，里面是具体的处理逻辑
- `return` → 把结果"吐出来"

### 写法二：函数表达式

函数也可以像一个值一样，赋给变量：

```js
const add = function(a, b) {
  return a + b;
};

console.log(add(3, 5));  // 输出：8
```

这里的 `function(a, b) { ... }` 是一个"匿名函数"（没有名字），然后把它存到了 `add` 这个变量里。

> 💡 函数声明和函数表达式的主要区别：函数声明会"提升"到作用域顶部，也就是说你可以在定义之前调用它；函数表达式不行。不过作为初学者，建议你**先定义，后调用**，这样不会出错。

### 写法三：箭头函数（最简洁）

箭头函数是 ES6 引入的新写法，更短更优雅：

```js
// 完整写法
const add = (a, b) => {
  return a + b;
};

// 如果函数体只有一行 return，可以进一步简写
const add = (a, b) => a + b;

console.log(add(3, 5));  // 输出：8
```

拆解箭头 `=>`：
- 左边 `(a, b)` 是参数
- 右边 `a + b` 是返回值
- 你可以把 `=>` 想象成"变成"：接收 `a, b`，变成 `a + b`

只有一个参数时，括号可以省略：

```js
const square = x => x * x;
console.log(square(4));  // 输出：16
```

没有参数时，必须写空括号：

```js
const sayHi = () => console.log("Hi!");
sayHi();  // 输出：Hi!
```

### 源码中的函数例子

Claude Code 的 `src/Task.ts` 中有一个 `generateTaskId` 函数，用来生成任务 ID：

```js
// 来自 src/Task.ts（去掉类型注解后的纯 JS 版本）
const TASK_ID_ALPHABET = '0123456789abcdefghijklmnopqrstuvwxyz'

function getTaskIdPrefix(type) {
  const TASK_ID_PREFIXES = {
    local_bash: 'b',
    local_agent: 'a',
    remote_agent: 'r',
    in_process_teammate: 't',
    local_workflow: 'w',
    monitor_mcp: 'm',
    dream: 'd',
  }
  return TASK_ID_PREFIXES[type] ?? 'x'
  // ?? 是"空值合并运算符"：如果左边是 null 或 undefined，就返回右边的值
}

function generateTaskId(type) {
  const prefix = getTaskIdPrefix(type)
  const bytes = randomBytes(8)
  let id = prefix
  for (let i = 0; i < 8; i++) {
    id += TASK_ID_ALPHABET[bytes[i] % TASK_ID_ALPHABET.length]
  }
  return id
}
```

逐行拆解 `generateTaskId`：
1. `function generateTaskId(type)` → 定义一个叫 `generateTaskId` 的函数，接收一个参数 `type`
2. `const prefix = getTaskIdPrefix(type)` → 先根据类型获取前缀字母
3. `const bytes = randomBytes(8)` → 生成 8 个随机字节（需要 `import { randomBytes } from 'crypto'`）
4. `let id = prefix` → 初始化 ID，先放前缀
5. `for (let i = 0; i < 8; i++)` → 循环 8 次
6. `id += TASK_ID_ALPHABET[...]` → 每次从字母表中取一个字符，拼接到 `id` 后面
7. `return id` → 返回生成的 ID

### 回调函数的概念

回调函数就是"把一个函数当作参数传给另一个函数"。

想象你去餐厅吃饭：你点完菜后，不会一直站在厨房门口等着，而是回到座位上。厨师做好菜后，"回调"（叫）服务员给你端上来。

```js
function cookDish(dishName, callback) {
  console.log(`正在做 ${dishName}...`);
  // 假设菜做好了
  callback(dishName);
}

function serveDish(dish) {
  console.log(`${dish} 做好了，请慢用！`);
}

cookDish("红烧肉", serveDish);
// 输出：
// 正在做 红烧肉...
// 红烧肉 做好了，请慢用！
```

这里 `serveDish` 就是回调函数——它不是立即执行的，而是等 `cookDish` 内部"菜做好了"之后才调用的。

回调函数在 JavaScript 中无处不在，尤其在处理异步操作（如网络请求、文件读取）时。

## 2.2 函数参数

### 可选参数

JavaScript 函数的参数本质上都是"可选的"——调用时可以不传，不传的那个参数在函数内部会变成 `undefined`：

```js
function greet(name, greeting) {
  console.log(name);      // 输出：Claude
  console.log(greeting);  // 输出：undefined（因为没传第二个参数）
}

greet("Claude");  // 只传了一个参数
```

这就是 TypeScript 中 `?`（可选参数）的底层原理：JavaScript 本身就允许不传参数，不传的就是 `undefined`。

### 默认参数

有时候调用函数时不想传所有参数，可以给参数设一个"默认值"：

```js
function greet(name, greeting = 'Hello') {
  return `${greeting}, ${name}!`;
}

console.log(greet("Claude"));           // 输出：Hello, Claude!
console.log(greet("Claude", "Hi"));     // 输出：Hi, Claude!
console.log(greet("Claude", "Good morning")); // 输出：Good morning, Claude!
```

拆解：
- `greeting = 'Hello'` → 如果调用时不传第二个参数，`greeting` 自动变成 `'Hello'`
- 这就像餐厅的套餐：如果你不指定饮料，默认给你一杯白开水

### 剩余参数（Rest Parameters）

有时候你不知道调用者会传多少个参数，用 `...` 可以把所有多余的参数收集成一个数组：

```js
function sum(...numbers) {
  let total = 0;
  for (const num of numbers) {
    total += num;
  }
  return total;
}

console.log(sum(1, 2, 3));     // 输出：6
console.log(sum(10, 20));      // 输出：30
console.log(sum());            // 输出：0
```

拆解 `...numbers`：
- `...` 是"剩余"的意思
- 调用 `sum(1, 2, 3)` 时，`numbers` 会变成 `[1, 2, 3]`

### 源码中的剩余参数例子

Claude Code 的 `src/ink/termio/osc.ts` 中有一个 `osc` 函数，用来生成终端控制序列：

```js
// 来自 src/ink/termio/osc.ts（去掉类型注解后的纯 JS 版本）
const OSC_PREFIX = '\x1b]'
const SEP = ';'
const BEL = '\x07'
const ST = '\x1b\\'

// env 是一个包含环境配置的对象（在真实源码中从外部传入）
const env = { terminal: 'default' }

function osc(...parts) {
  const terminator = env.terminal === 'kitty' ? ST : BEL
  return OSC_PREFIX + parts.join(SEP) + terminator
}

// 调用示例
console.log(osc('1337', 'File=name'));
// 输出（不可见字符已转义）：\x1b]1337;File=name\x07
```

拆解：
- `...parts` → 收集所有传入的参数为一个数组
- `parts.join(SEP)` → 用分号 `;` 把所有参数连接成字符串
- 这个函数可以接受任意多个参数：2 个、3 个、10 个都可以

### 参数解构

当函数接收一个对象参数时，可以直接在参数列表中解构：

```js
// 普通写法
function printUser(user) {
  console.log(user.name);
  console.log(user.age);
}

// 解构写法（更简洁）
function printUser({ name, age }) {
  console.log(name);
  console.log(age);
}

printUser({ name: "Claude", age: 3 });
// 输出：
// Claude
// 3
```

## 2.3 对象进阶

### 对象字面量简写

当属性名和变量名相同时，可以简写：

```js
const name = "Claude";
const age = 3;

// 普通写法
const person1 = {
  name: name,
  age: age
};

// 简写
const person2 = {
  name,
  age
};

console.log(person2);  // 输出：{ name: "Claude", age: 3 }
```

### 方法简写

对象中的函数可以省去 `: function`：

```js
// 普通写法
const calculator1 = {
  add: function(a, b) {
    return a + b;
  }
};

// 简写
const calculator2 = {
  add(a, b) {
    return a + b;
  }
};

console.log(calculator2.add(2, 3));  // 输出：5
```

### 计算属性名

属性名可以是动态的（用方括号包起来）：

```js
const key = "name";

const person = {
  [key]: "Claude",
  ["age"]: 3
};

console.log(person);  // 输出：{ name: "Claude", age: 3 }
```

这在需要根据变量生成属性名时非常有用。

### 源码中的对象结构

Claude Code 的 `src/Task.ts` 中定义了任务对象的结构：

```js
// 来自 src/Task.ts（去掉类型注解后的纯 JS 简化版）
const task = {
  name: "执行 bash 命令",
  type: "local_bash",
  
  // 方法简写
  kill(taskId, setAppState) {
    console.log(`正在终止任务 ${taskId}`);
    // ... 实际的终止逻辑
  },

  // 方法简写也可以有返回值
  getInfo() {
    // this 指向调用这个方法的对象，也就是 task
    return {
      name: this.name,  // 等价于 task.name
      status: "running"
    }
  }
}

task.kill("task_001");  // 调用方法
```

> 💡 **`this` 是什么？** 在对象的方法里，`this` 指向"调用这个方法的对象"。比如 `task.getInfo()` 中，`this` 就是 `task`。所以 `this.name` 就是 `"执行 bash 命令"`。可以把 `this` 理解为"我自己"。

## 2.4 解构赋值

解构赋值是一种"批量提取"的语法，可以从对象或数组中快速取出值。

### 对象解构

```js
const person = {
  name: "Claude",
  age: 3,
  isAdmin: false
};

// 普通写法
const name1 = person.name;
const age1 = person.age;

// 解构写法（更简洁）
const { name, age } = person;

console.log(name);  // 输出：Claude
console.log(age);   // 输出：3
```

还可以设置默认值：

```js
const options = {
  showValues: true
};

const { showValues = true, hideFunctions = false, themeName = 'dark' } = options;

console.log(showValues);     // 输出：true（来自对象）
console.log(hideFunctions);  // 输出：false（默认值，对象中没有）
console.log(themeName);      // 输出：dark（默认值，对象中没有）
```

### 数组解构

```js
const colors = ["red", "green", "blue"];

const [first, second] = colors;

console.log(first);   // 输出：red
console.log(second);  // 输出：green
```

还可以跳过某些元素：

```js
const [first, , third] = colors;
console.log(third);   // 输出：blue
```

### 嵌套解构

```js
const user = {
  name: "Claude",
  address: {
    city: "San Francisco",
    country: "USA"
  }
};

const { name, address: { city } } = user;

console.log(name);  // 输出：Claude
console.log(city);  // 输出：San Francisco
```

### 源码中的解构例子

Claude Code 的 `src/utils/treeify.ts` 中大量使用了解构：

```js
// 来自 src/utils/treeify.ts（去掉类型注解后的纯 JS 版本）
function treeify(obj, options = {}) {
  const {
    showValues = true,
    hideFunctions = false,
    themeName = 'dark',
    treeCharColors = {},
  } = options

  const lines = []
  // ... 后续代码
}
```

拆解：
- `options = {}` → 如果调用时不传 `options`，默认是一个空对象
- `const { showValues = true, ... } = options` → 从 `options` 中解构出各个配置项，并设置默认值
- 这样写比 `const showValues = options.showValues ?? true` 简洁多了

## 2.5 展开运算符（Spread）

展开运算符 `...` 可以把一个对象或数组"拆开"，然后合并到新的对象或数组中。

### 对象展开

```js
const defaults = {
  theme: "dark",
  fontSize: 14,
  showLineNumbers: true
};

const userSettings = {
  fontSize: 16  // 用户只改了字体大小
};

// 合并：后面的属性会覆盖前面的同名属性
const finalSettings = {
  ...defaults,      // 先把所有默认设置展开
  ...userSettings   // 再把用户设置展开，同名属性会覆盖默认值
};

console.log(finalSettings);
// 输出：{ theme: "dark", fontSize: 16, showLineNumbers: true }
```

### 数组展开

```js
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

const combined = [...arr1, ...arr2];
console.log(combined);  // 输出：[1, 2, 3, 4, 5, 6]

// 还可以在中间插入元素
const mixed = [0, ...arr1, 99, ...arr2];
console.log(mixed);  // 输出：[0, 1, 2, 3, 99, 4, 5, 6]
```

### 函数调用展开

```js
const numbers = [10, 5, 8];

// 普通写法
console.log(Math.max(numbers[0], numbers[1], numbers[2]));  // 输出：10

// 用展开运算符
console.log(Math.max(...numbers));  // 输出：10
```

这里 `...numbers` 把数组 `[10, 5, 8]` 展开成了 `10, 5, 8` 三个独立参数。

### 展开运算符与剩余参数的对比

它们长得一样（都是 `...`），但用法相反：

| 场景 | 作用 | 比喻 |
|------|------|------|
| 剩余参数 `...args` | 把多个参数**收集**成数组 | 把散落一地的积木装进盒子 |
| 展开运算符 `...arr` | 把数组**展开**成多个值 | 把盒子里的积木倒出来 |

```js
// 收集（剩余参数）
function collect(...args) {
  console.log(args);  // [1, 2, 3]
}
collect(1, 2, 3);

// 展开（展开运算符）
const arr = [1, 2, 3];
console.log(...arr);  // 等价于 console.log(1, 2, 3)
```

## 2.6 本章小结

| 概念 | 含义 | 例子 |
|------|------|------|
| 函数声明 | 最常见的函数写法 | `function add(a, b) { return a + b; }` |
| 函数表达式 | 把函数赋给变量 | `const add = function(a, b) { ... }` |
| 箭头函数 | 最简洁的函数写法 | `const add = (a, b) => a + b` |
| 可选参数 | 不传时默认为 undefined | `function greet(name, greeting)` 可只传一个参数 |
| 默认参数 | 参数不传时的默认值 | `function greet(name, greeting = 'Hello')` |
| 剩余参数 | 收集多余参数为数组 | `function fn(...args)` |
| 参数解构 | 直接从对象参数中提取 | `function fn({ name, age })` |
| 对象简写 | 属性名和变量名相同时省略 | `{ name, age }` |
| 方法简写 | 省略 `: function` | `{ greet() { ... } }` |
| 计算属性名 | 动态属性名 | `{ [key]: value }` |
| 对象解构 | 从对象中提取属性 | `const { a, b } = obj` |
| 数组解构 | 从数组中提取元素 | `const [a, b] = arr` |
| 对象展开 | 合并对象 | `{ ...obj1, ...obj2 }` |
| 数组展开 | 合并数组 | `[ ...arr1, ...arr2 ]` |
| 函数调用展开 | 数组变参数 | `fn(...args)` |
