# 第三章：条件判断与类型检查

> 本章目标：学会让代码"根据不同情况做出不同决定"，以及如何检查一个值到底是什么类型。
>
> 想象你是一个餐厅服务员：顾客点牛排，你上牛排；顾客点沙拉，你上沙拉。代码也一样，需要根据条件做不同的事情。

---

## 3.1 if/else 语句

### 基本语法

`if` 语句就像生活中的"如果...就..."：

```js
const age = 18;

if (age >= 18) {
  console.log('你可以考驾照了！');
}
```

如果条件成立（`true`），就执行大括号里的代码；如果不成立（`false`），就跳过。

加上 `else`，可以处理"否则"的情况：

```js
const age = 16;

if (age >= 18) {
  console.log('你可以考驾照了！');
} else {
  console.log('再等两年吧。');
}
```

多个条件用 `else if`：

```js
const score = 85;

if (score >= 90) {
  console.log('优秀');
} else if (score >= 80) {
  console.log('良好');
} else if (score >= 60) {
  console.log('及格');
} else {
  console.log('不及格');
}
```

> 💡 **类比**：`if/else if/else` 就像自动售货机——按不同按钮，掉出不同饮料。

---

### 比较运算符

条件判断离不开比较。JavaScript 提供这些比较运算符：

| 运算符 | 含义 | 例子 | 结果 |
|--------|------|------|------|
| `===` | 严格相等 | `5 === 5` | `true` |
| `!==` | 严格不相等 | `5 !== 3` | `true` |
| `>` | 大于 | `10 > 5` | `true` |
| `<` | 小于 | `3 < 7` | `true` |
| `>=` | 大于等于 | `5 >= 5` | `true` |
| `<=` | 小于等于 | `4 <= 5` | `true` |

#### ⚠️ 为什么总是用 `===` 而不是 `==`？

`===` 是**严格相等**，不仅比较值，还比较类型。`==` 是**松散相等**，会在比较前偷偷转换类型，经常出 bug。

```js
// 松散相等 == 的陷阱
console.log(0 == '');      // true  !?  0 和空字符串被认为相等
console.log(0 == '0');     // true  !?  字符串 '0' 被转成了数字 0
console.log(false == 0);   // true  !?  false 被转成了 0
console.log(null == undefined); // true

// 严格相等 === 更安全
console.log(0 === '');     // false ✅ 类型不同，直接返回 false
console.log(0 === '0');    // false ✅
console.log(false === 0);  // false ✅
```

> 🎯 **黄金法则**：写代码时永远用 `===` 和 `!==`，不要用 `==` 和 `!=`。

---

### 逻辑运算符

有时候一个条件不够，需要组合多个条件：

#### `&&`（与 / AND）

两边都为 `true`，结果才为 `true`：

```js
const hasId = true;
const isAdult = true;

if (hasId && isAdult) {
  console.log('可以进场');
}
```

> 💡 **类比**：`&&` 就像进小区门——需要同时刷门禁卡 **并且** 人脸识别通过。

#### `||`（或 / OR）

只要有一边为 `true`，结果就为 `true`：

```js
const isWeekend = true;
const isHoliday = false;

if (isWeekend || isHoliday) {
  console.log('今天不用上班');
}
```

> 💡 **类比**：`||` 就像开门方式——刷门禁卡 **或者** 输密码，都能进门。

#### `!`（非 / NOT）

取反：`true` 变 `false`，`false` 变 `true`：

```js
const isRaining = false;

if (!isRaining) {
  console.log('不用带伞');
}
```

### 源码例子：判断任务是否结束

在 Claude Code 的 `src/Task.ts` 中，有一个判断任务是否进入终态的函数：

```js
// 来自 src/Task.ts（转换为纯 JavaScript）
export function isTerminalTaskStatus(status) {
  return status === 'completed' || status === 'failed' || status === 'killed'
}

// 使用
console.log(isTerminalTaskStatus('completed')); // true
console.log(isTerminalTaskStatus('running'));   // false
console.log(isTerminalTaskStatus('killed'));    // true
```

**逐行拆解**：

1. `status === 'completed'` —— 状态是 "已完成" 吗？
2. `||` —— 或者...
3. `status === 'failed'` —— 状态是 "失败" 吗？
4. `||` —— 或者...
5. `status === 'killed'` —— 状态是 "被终止" 吗？
6. 只要满足任意一个，任务就是"终态"，不会再变化了。

## 3.2 switch 语句

### 基本语法

当你需要对同一个变量做多次等值判断时，`switch` 比一堆 `if/else if` 更清晰：

```js
const fruit = 'apple';

switch (fruit) {
  case 'apple':
    console.log('苹果，5元一斤');
    break;
  case 'banana':
    console.log('香蕉，3元一斤');
    break;
  case 'orange':
    console.log('橙子，4元一斤');
    break;
  default:
    console.log('不知道这是什么水果');
}
```

**工作原理**：
1. 把 `fruit` 的值和每个 `case` 后面的值比较
2. 匹配到哪个 `case`，就从那里开始执行
3. `break` 表示"到此为止，跳出 switch"
4. `default` 是"如果以上都不匹配"的兜底

### break 的重要性

**忘记写 `break` 会导致"穿透"（fall-through）**：

```js
const grade = 'B';

switch (grade) {
  case 'A':
    console.log('优秀');
    // 没有 break，继续往下执行！
  case 'B':
    console.log('良好');
    // 没有 break，继续往下执行！
  case 'C':
    console.log('及格');
    break;
}
// 输出：
// 良好
// 及格
```

有时候可以利用这个特性让多个 `case` 共享同一段代码：

```js
const day = 'Saturday';

switch (day) {
  case 'Saturday':
  case 'Sunday':
    console.log('周末愉快！');
    break;
  case 'Monday':
  case 'Tuesday':
  case 'Wednesday':
  case 'Thursday':
  case 'Friday':
    console.log('工作日加油！');
    break;
  default:
    console.log('这不是一个有效的日期');
}
```

### 什么时候用 switch，什么时候用 if？

| 场景 | 推荐 | 原因 |
|------|------|------|
| 同一个变量的多个等值判断 | `switch` | 更清晰 |
| 范围判断（如 `score >= 60`）| `if` | switch 做不到 |
| 复杂条件组合 | `if` | 更灵活 |

## 3.3 typeof 运算符

有时候你不知道一个变量里面是什么，需要检查一下它的类型。`typeof` 就是干这个的：

```js
console.log(typeof 'hello');      // 'string'
console.log(typeof 42);           // 'number'
console.log(typeof true);         // 'boolean'
console.log(typeof undefined);    // 'undefined'
console.log(typeof {});           // 'object'
console.log(typeof []);           // 'object'  （数组也是 object！）
console.log(typeof function(){}); // 'function'
```

> 💡 **类比**：`typeof` 就像快递单上的"物品类型"栏——让你知道包裹里是衣服、电子产品还是食品。

### 检查基本类型

```js
function process(value) {
  if (typeof value === 'string') {
    console.log('是个字符串，长度：' + value.length);
  } else if (typeof value === 'number') {
    console.log('是个数字，两倍：' + value * 2);
  } else {
    console.log('是其他类型');
  }
}

process('hello'); // 是个字符串，长度：5
process(10);      // 是个数字，两倍：20
```

### typeof 的陷阱：null

JavaScript 有个著名的历史 bug：

```js
console.log(typeof null);  // 'object'  !?
```

`null` 明明是"空值"，但 `typeof null` 返回 `'object'`。这是 JavaScript 诞生时的设计缺陷，永远无法修复了。

所以如果你要判断一个值是不是 `null`，不能靠 `typeof`：

```js
const value = null;

// ❌ 错误方式
if (typeof value === 'object') {
  console.log('可能是对象，也可能是 null');
}

// ✅ 正确方式
if (value === null) {
  console.log('确实是 null');
}
```

### 源码例子：treeify.ts 中的 typeof 判断

在 Claude Code 的 `src/utils/treeify.ts` 中，有一个把对象转成树形字符串的函数。它需要判断当前节点是字符串还是对象：

```js
// 来自 src/utils/treeify.ts（简化为纯 JavaScript）
function growBranch(node, prefix, _isLast, depth = 0) {
  if (typeof node === 'string') {
    // node 在这里确定是一个字符串
    lines.push(prefix + colorize(node, treeCharColors.value));
    return;
  }

  if (typeof node !== 'object' || node === null) {
    // node 在这里是 null 或其他原始类型
    if (showValues) {
      const valueStr = String(node);
      lines.push(prefix + colorize(valueStr, treeCharColors.value));
    }
    return;
  }

  // 到这里，node 确定是一个对象（不是 null）
  // 继续处理对象的每个属性...
}
```

**逐行拆解**：

1. `if (typeof node === 'string')` —— 如果节点是字符串，直接显示它，不需要继续递归
2. `if (typeof node !== 'object' || node === null)` —— 如果不是对象，或者是 null，也直接显示
3. 注意第二个条件用了 `|| node === null`，就是为了处理 `typeof null === 'object'` 这个 bug！
4. 通过这两层过滤，剩下的 `node` 一定是真正的对象，可以安全地访问它的属性

## 3.4 in 运算符

`in` 运算符用来检查一个对象是否有某个属性：

```js
const person = {
  name: '小明',
  age: 20,
};

console.log('name' in person);   // true
console.log('age' in person);    // true
console.log('height' in person); // false
```

> 💡 **类比**：`in` 就像查字典——"这个词在字典里吗？"

### 和点运算符的区别

```js
const obj = { a: 1, b: undefined };

console.log(obj.a);        // 1
console.log(obj.b);        // undefined
console.log(obj.c);        // undefined（不存在的属性也返回 undefined）

console.log('a' in obj);   // true
console.log('b' in obj);   // true（值为 undefined，但属性存在）
console.log('c' in obj);   // false（属性不存在）
```

`in` 能区分"属性存在但值为 undefined"和"属性不存在"，而点运算符做不到。

### 源码例子：检查对象是否有某个属性

在 Claude Code 的源码中，经常用 `in` 来判断数据结构符合哪种格式：

```js
// 来自 src/bridge/bridgeMessaging.js（概念转换）
function isControlRequest(value) {
  return (
    typeof value === 'object' &&
    value !== null &&
    'type' in value &&
    value.type === 'control_request' &&
    'request_id' in value &&
    'request' in value
  );
}

// 使用
const msg1 = { type: 'control_request', request_id: '123', request: 'ping' };
const msg2 = { type: 'control_response', response: 'pong' };

console.log(isControlRequest(msg1)); // true
console.log(isControlRequest(msg2)); // false
```

**逐行拆解**：

1. `typeof value === 'object'` —— 先确保是对象
2. `value !== null` —— 排除 null
3. `'type' in value` —— 对象有 `type` 属性吗？
4. `value.type === 'control_request'` —— `type` 的值是 `'control_request'` 吗？
5. `'request_id' in value` —— 有 `request_id` 属性吗？
6. `'request' in value` —— 有 `request` 属性吗？

这样一步步检查，确保传入的对象确实符合"控制请求"的格式。

> 💡 **核心思想**：通过 `typeof`、`in` 等检查，我们可以一步步"缩小"变量的可能性范围。这就是 TypeScript 中**类型收窄（Narrowing）**的底层逻辑——在 JavaScript 中，虽然类型是动态的，但检查之后你心里已经清楚它是什么了。

## 3.5 自定义类型检查函数

你可以把类型检查逻辑封装成函数，方便复用：

```js
function isString(value) {
  return typeof value === 'string';
}

function isNonNullObject(value) {
  return typeof value === 'object' && value !== null;
}

console.log(isString('hello'));  // true
console.log(isString(42));       // false
console.log(isNonNullObject({})); // true
console.log(isNonNullObject(null)); // false
```

在 TypeScript 中，这种返回 `boolean` 的自定义检查函数可以升级为**类型守卫**（`x is Type`），让编译器也能理解你的检查逻辑。但在 JavaScript 中，它们就是普通的辅助函数，帮助你保持代码整洁。

## 3.6 通过属性值判断对象结构

在 JavaScript 中，一个对象通常会有一个"类型标识"属性（比如 `type`、`kind`），用来告诉你这个对象长什么样：

```js
const message1 = { type: 'text', content: '你好' };
const message2 = { type: 'image', url: 'http://example.com/cat.jpg' };

function handleMessage(msg) {
  if (msg.type === 'text') {
    // 到这里，我们知道 msg 一定有 content 属性
    console.log('文本消息：' + msg.content);
  } else if (msg.type === 'image') {
    // 到这里，我们知道 msg 一定有 url 属性
    console.log('图片地址：' + msg.url);
  }
}

handleMessage(message1); // 文本消息：你好
handleMessage(message2); // 图片地址：http://example.com/cat.jpg
```

这就是 TypeScript 中**可辨识联合（Discriminated Union）**的底层模式：对象的某个属性（通常是 `type`）充当了"标签"，让你可以根据它的值来判断对象有哪些其他属性。在 JavaScript 中，这种模式同样非常常见和实用。

## 3.7 三元运算符

三元运算符是 `if/else` 的简写形式，适合简单的条件赋值：

```js
const age = 20;
const status = age >= 18 ? '成年人' : '未成年人';

console.log(status); // '成年人'
```

语法结构：

```
条件 ? 条件为真时的值 : 条件为假时的值
```

> 💡 **类比**：三元运算符就像自动贩卖机的简写——投够钱了？出货 ：退币。

### 和 if/else 的对比

```js
// if/else 写法
let greeting;
if (hour < 12) {
  greeting = '早上好';
} else {
  greeting = '下午好';
}

// 三元运算符写法（更简洁）
const greeting = hour < 12 ? '早上好' : '下午好';
```

### 嵌套三元运算符（不推荐）

三元运算符可以嵌套，但超过一层就很难读了：

```js
const score = 85;

// 一层嵌套还能接受
const grade = score >= 90 ? 'A' : score >= 60 ? 'B' : 'C';

// ❌ 多层嵌套——很难读懂！
const result = a > 0 ? (b > 0 ? 'a和b都大于0' : 'a大于0，b不大于0') : (b > 0 ? 'a不大于0，b大于0' : 'a和b都不大于0');
```

> 🎯 **建议**：嵌套三元运算符最多一层。更复杂的情况请用 `if/else`。

## 3.8 短路求值

`&&` 和 `||` 有个重要特性叫**短路求值**——如果左边的结果已经能决定最终结果，右边就不会执行了。

> 💡 **重要**：`&&` 和 `||` 返回的不一定是 `true` 或 `false`，而是**能决定结果的那个操作数本身**。这在 JavaScript 中被广泛用于设置默认值和安全访问。

### `&&` 的短路行为

```js
// &&：左边为假，右边不执行
const a = false;
const b = true;

if (a && b) {
  // 不会进入这里，因为 a 是 false，JavaScript 根本不去看 b
}
```

利用这个特性可以做"安全访问"：

```js
const user = { name: '小明' };

// 如果 user 存在，才访问 user.name
const userName = user && user.name;
console.log(userName); // '小明'（返回 user.name 的值）

const nullUser = null;
const nullName = nullUser && nullUser.name;
console.log(nullName); // null（返回 nullUser 的值，右边没执行）
```

如果 `nullUser` 是 `null`，`nullUser.name` 本来会报错（不能从 null 读取属性）。但因为 `&&` 短路了，右边根本没执行，所以安全返回 `null`。

### `||` 的短路行为

```js
// ||：左边为真，右边不执行
const x = 'hello';
const y = 'world';

const result = x || y;
// x 是 'hello'（真值），所以直接返回 'hello'，不会去看 y
```

利用这个特性可以设置默认值：

```js
function greet(name) {
  // 如果 name 是"假值"（undefined、空字符串、0、false、null、NaN），就用 '陌生人'
  const actualName = name || '陌生人';
  console.log('你好，' + actualName);
}

greet('小明');     // 你好，小明
greet('');         // 你好，陌生人
greet(undefined);  // 你好，陌生人
```

> ⚠️ 注意：在 ES2020 之后，更推荐使用 `??`（空值合并运算符）来设置默认值，因为它只会在 `null` 或 `undefined` 时生效，不会把 `0`、`''`、`false` 也当成"空"。不过 `||` 的短路原理你仍然需要理解。

## 3.9 严格相等 vs 松散相等（再强调）

本章开头已经提过，但因为太重要了，值得再讲一遍。

### `===` 和 `==` 的区别

| | `===`（严格相等）| `==`（松散相等）|
|---|---|---|
| 比较前转换类型 | ❌ 不转换 | ✅ 会转换 |
| `0 === ''` | `false` | `true` |
| `0 === false` | `false` | `true` |
| `null === undefined` | `false` | `true` |
| `NaN === NaN` | `false` | `false` |

### Object.is() —— 更严格的比较

`Object.is()` 和 `===` 几乎一样，但有两个特例：

```js
// === 认为 NaN 不等于 NaN
console.log(NaN === NaN);        // false

// Object.is() 认为 NaN 等于 NaN
console.log(Object.is(NaN, NaN)); // true

// === 认为 +0 等于 -0
console.log(+0 === -0);          // true

// Object.is() 认为 +0 不等于 -0
console.log(Object.is(+0, -0));  // false
```

日常开发中 `===` 够用了。遇到 `NaN` 比较时才需要考虑 `Object.is()`。

## 本章小结

| 概念 | 写法 | 作用 |
|------|------|------|
| if/else | `if (条件) { ... } else { ... }` | 根据条件执行不同代码 |
| 严格相等 | `===` | 值和类型都相同才为 true |
| 严格不等 | `!==` | 值或类型不同就为 true |
| 逻辑与 | `&&` | 两边都为 true 才为 true |
| 逻辑或 | `\|\|` | 一边为 true 就为 true |
| 逻辑非 | `!` | 取反 |
| switch | `switch (x) { case 'a': ... }` | 多分支等值判断 |
| typeof | `typeof x` | 返回值的类型字符串 |
| in | `'prop' in obj` | 检查对象是否有某属性 |
| 自定义类型检查 | `function isX(v) { ... }` | 封装判断逻辑，保持代码整洁 |
| 属性值判断结构 | `if (obj.type === 'A')` | 通过标签属性区分对象形状 |
| 三元运算符 | `条件 ? A : B` | if/else 的简写 |
| 短路求值 | `a && b`、`a \|\| b` | 左边能决定结果时跳过右边 |
| Object.is | `Object.is(a, b)` | 比 === 更严格的比较 |

**核心原则**：
1. 永远用 `===` 和 `!==`，不用 `==` 和 `!=`
2. `typeof null === 'object'` 是 bug，判断 null 用 `=== null`
3. `switch` 别忘了 `break`
4. 嵌套三元运算符最多一层
5. 通过 `typeof`、`in`、属性值判断可以"缩小"变量的类型范围——这是 TypeScript 类型收窄的底层逻辑

### 练习 2

补全下面的函数，用 `typeof` 和 `in` 判断传入的值是否符合预期格式：

```js
function isValidPerson(value) {
  // 要求：value 是对象（不是 null），且有 name 属性（字符串）和 age 属性（数字）
  // 请补全代码
}

console.log(isValidPerson({ name: '小明', age: 20 })); // 应输出 true
console.log(isValidPerson({ name: '小红' }));          // 应输出 false
console.log(isValidPerson(null));                       // 应输出 false
console.log(isValidPerson({ name: 123, age: 20 }));    // 应输出 false
```
