> 本章目标：理解 JavaScript 中 `class` 的基本语法，能看懂 Claude Code 源码中的自定义错误类和其他类的用法。

## 8.1 什么是面向对象？

### 生活中的类比

想象你要造汽车：

- **类（Class）** 就像汽车的**设计蓝图**。蓝图上画了汽车该有的东西——几个轮子、什么颜色、怎么发动。
- **实例（Instance）** 就是根据蓝图造出来的**具体汽车**。你可以用同一张蓝图造出很多辆汽车，每辆都有自己的颜色、里程数。

**面向对象编程（OOP）** 的核心思想是：把数据和操作数据的方法打包在一起，形成一个"对象"。

### 对象 vs 类

```js
// 一个普通的对象
const car1 = {
  brand: 'Toyota',
  color: 'red',
  drive() {
    console.log('开车了！');
  },
};

// 另一个普通的对象
const car2 = {
  brand: 'Honda',
  color: 'blue',
  drive() {
    console.log('开车了！');
  },
};
```

上面的写法很重复。如果我们要造 100 辆车，每辆都要手写一遍，而且如果 `drive` 方法要改，100 辆车都要改。

**类**就是来解决这个问题的：定义一次，重复使用。

## 8.2 class 基本语法

### 定义一个类

```js
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}

// 创建实例
const alice = new Person('Alice', 25);
const bob = new Person('Bob', 30);

console.log(alice.name);  // 输出：Alice
console.log(bob.age);     // 输出：30
```

拆解：
- `class Person` → 定义一个名叫 `Person` 的类
- `constructor(name, age)` → **构造函数**，当你用 `new` 创建实例时，会自动调用这个函数
- `this.name = name` → `this` 指向**当前正在创建的那个实例**，给实例添加属性
- `new Person('Alice', 25)` → 创建一个新的 `Person` 实例

### this 关键字

`this` 是一个特殊的关键词，它的意思是"**当前这个对象**"：

```js
class Dog {
  constructor(name) {
    this.name = name;  // this 指向新建的狗对象
  }
}

const myDog = new Dog('旺财');
// myDog 就是 { name: '旺财' }
```

`this` 的值取决于函数是怎么被调用的。在构造函数里，`this` 指向新创建的实例。

## 8.3 方法

类里面除了属性，还可以定义**方法**（函数）：

```js
class Person {
  constructor(name) {
    this.name = name;
  }

  // 实例方法
  sayHello() {
    console.log(`你好，我叫 ${this.name}`);
  }

  walk() {
    console.log(`${this.name} 正在走路`);
  }
}

const alice = new Person('Alice');
alice.sayHello();  // 输出：你好，我叫 Alice
alice.walk();      // 输出：Alice 正在走路
```

拆解：
- `sayHello()` 和 `walk()` 是定义在类里的方法
- 调用时通过实例名加点调用：`alice.sayHello()`
- 方法内部用 `this.name` 访问当前实例的属性

### Claude Code 源码例子：自定义 Error 类（简化版）

```js
// 来自 src/bridge/bridgeApi.ts（简化版）
class BridgeFatalError extends Error {
  constructor(message, status, errorType) {
    super(message);              // 调用父类 Error 的构造函数
    this.name = 'BridgeFatalError';  // 设置错误名称
    this.status = status;        // 自定义属性
    this.errorType = errorType;  // 自定义属性
  }

  // 实例方法：获取错误摘要
  getSummary() {
    return `${this.name}: ${this.message} (status=${this.status})`;
  }
}

// 使用
const err = new BridgeFatalError('连接失败', 401, 'unauthorized');
console.log(err.message);      // 连接失败
console.log(err.status);       // 401
console.log(err.getSummary()); // BridgeFatalError: 连接失败 (status=401)
```

拆解：
- `extends Error` → 继承自 JavaScript 内置的 `Error` 类
- `super(message)` → 调用父类的构造函数，设置错误消息
- `this.status = status` → 给实例添加额外属性
- `getSummary()` → 自定义方法，返回格式化的错误信息

## 8.4 继承（extends）

继承的意思是：子类可以**复用**父类的代码，同时还能添加自己的特性。

就像生物遗传：你继承了父母的一些特征（眼睛颜色、血型），同时又有自己的独特之处（性格、爱好）。

### 基本语法

```js
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} 发出了声音`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name);        // 调用父类的构造函数
    this.breed = breed; // 子类自己的属性
  }

  speak() {
    super.speak();      // 调用父类的 speak 方法
    console.log(`${this.name} 汪汪叫！`);
  }
}

const dog = new Dog('旺财', '柴犬');
dog.speak();
```

输出：
```
旺财 发出了声音
旺财 汪汪叫！
```

拆解：
- `class Dog extends Animal` → `Dog` 继承自 `Animal`
- `super(name)` → 在子类的构造函数中，**必须先调用** `super()`，它会执行父类的构造函数
- `super.speak()` → 调用父类的 `speak` 方法
- 子类可以**重写**（override）父类的方法，实现自己的版本

### Claude Code 源码例子：BridgeFatalError

```js
// 来自 src/bridge/bridgeApi.ts（简化版）
class BridgeFatalError extends Error {
  constructor(message, status, errorType) {
    super(message);
    this.name = 'BridgeFatalError';
    this.status = status;
    this.errorType = errorType;
  }
}
```

这里 `BridgeFatalError` 继承了内置的 `Error` 类：
- `Error` 类本来就有 `message`、`name`、`stack` 等属性
- `BridgeFatalError` 添加了 `status` 和 `errorType` 属性
- 这样抛出的错误既有 `Error` 的基本功能，又有额外的上下文信息

> 💡 **TS 的 `override` 关键字**：在 TypeScript 中，如果你重写（覆盖）了父类的方法，推荐加上 `override` 关键字。它的作用是：如果父类实际上没有这个方法，TypeScript 会报错，防止你拼错方法名。**JavaScript 中没有 `override` 关键字**——子类方法会自动覆盖父类的同名方法，不需要额外声明。TypeScript 加上 `override` 是为了在编译时给你多一层保护。

### Claude Code 源码例子：McpToolCallError

```js
// 来自 src/services/mcp/client.ts（简化版）
class McpToolCallError extends Error {
  constructor(message, code) {
    super(message);
    this.name = 'McpToolCallError';
    this.code = code;  // 错误代码
  }
}

// 使用
try {
  callMcpTool('someTool');
} catch (error) {
  if (error instanceof McpToolCallError) {
    console.error(`工具调用失败，错误码：${error.code}`);
  }
}
```

## 8.5 getter 和 setter

有时候我们希望属性被读取或设置时，能**自动做一些事情**。getter 和 setter 就是干这个的。

```js
class Circle {
  constructor(radius) {
    this._radius = radius;  // 用下划线前缀表示"私有"（约定俗成）
  }

  // getter：读取 radius 时自动调用
  get radius() {
    console.log('有人读取了半径');
    return this._radius;
  }

  // setter：设置 radius 时自动调用
  set radius(value) {
    if (value <= 0) {
      throw new Error('半径必须大于 0');
    }
    console.log('有人设置了半径');
    this._radius = value;
  }

  // getter：计算属性
  get area() {
    return Math.PI * this._radius * this._radius;
  }
}

const c = new Circle(5);
console.log(c.radius);  // 输出：有人读取了半径 / 5
c.radius = 10;          // 输出：有人设置了半径
console.log(c.area);    // 输出：314.159...
```

拆解：
- `get radius()` → 定义 getter，访问 `c.radius` 时会自动执行这个方法
- `set radius(value)` → 定义 setter，给 `c.radius` 赋值时会自动执行这个方法
- `get area()` → 计算属性，每次访问都会重新计算

注意：getter/setter 和普通属性同名时，不会冲突，JS 会自动区分。

## 8.6 静态方法（static）

静态方法是**属于类本身**的方法，而不是属于某个实例的。

就像"人类的平均身高"是属于"人类"这个物种的，不属于某一个人。

```js
class MathUtils {
  // 静态方法
  static add(a, b) {
    return a + b;
  }

  static multiply(a, b) {
    return a * b;
  }
}

// 直接通过类名调用，不需要 new
console.log(MathUtils.add(2, 3));       // 5
console.log(MathUtils.multiply(4, 5));  // 20

// 不能通过实例调用
const utils = new MathUtils();
// utils.add(1, 2);  // ❌ 报错：utils.add is not a function
```

静态方法常用于：
- 工具函数（不需要实例数据）
- 工厂方法（用静态方法创建实例）

```js
class Person {
  constructor(name) {
    this.name = name;
  }

  // 工厂方法：创建一个匿名用户
  static createAnonymous() {
    return new Person('匿名用户');
  }
}

const anon = Person.createAnonymous();
console.log(anon.name);  // 匿名用户
```

## 8.7 私有字段（#field）

现代 JavaScript 提供了一种真正的私有属性语法：在属性名前加 `#`。

```js
class BankAccount {
  #balance = 0;  // 私有字段，类外部无法直接访问

  constructor(initialBalance) {
    this.#balance = initialBalance;
  }

  deposit(amount) {
    if (amount > 0) {
      this.#balance += amount;
    }
  }

  getBalance() {
    return this.#balance;
  }
}

const account = new BankAccount(100);
account.deposit(50);
console.log(account.getBalance());  // 150

// console.log(account.#balance);  // ❌ 语法错误：私有字段不能在类外部访问
```

拆解：
- `#balance` → 私有字段，只能在类的内部访问
- 这提供了真正的封装，防止外部代码随意修改内部状态

注意：私有字段是较新的语法（ES2022），在较旧的运行环境中可能需要转译。

## 8.8 class 与原型链的关系（简要提及）

JavaScript 的 `class` 本质上是一种**语法糖**，底层仍然是**原型链**。

初学者不需要深入理解原型链，只需要知道两个关键操作：

### instanceof 运算符

`instanceof` 用来检查一个对象是否是某个类的实例：

```js
class Animal {}
class Dog extends Animal {}

const dog = new Dog();

console.log(dog instanceof Dog);     // true
console.log(dog instanceof Animal);  // true（因为 Dog 继承自 Animal）
console.log(dog instanceof Object);  // true（所有对象都继承自 Object）
```

在 Claude Code 中经常看到这样的用法：

```js
// 来自 src/bridge/replBridge.ts（简化版）
try {
  await pollForWork();
} catch (err) {
  if (err instanceof BridgeFatalError) {
    console.error('桥接致命错误，停止轮询');
    shutdown();
  } else {
    console.error('普通错误，可以重试：', err.message);
  }
}
```

这里用 `instanceof` 判断错误的类型，从而决定不同的处理策略。

## 8.9 JSX 与组件的 JavaScript 本质

### 什么是 JSX？

JSX（JavaScript XML）是一种**语法扩展**，允许你在 JavaScript 中写类似 HTML 的代码：

```jsx
function Hello() {
  return <div>Hello, Claude Code!</div>;
}
```

### 组件就是函数！

**最重要的理解**：在上面的代码中，`Hello` 本质上就是一个**普通的 JavaScript 函数**！JSX 只是让这个函数返回的内容看起来更像 HTML 而已。

转译后，它变成了纯粹的 JavaScript：

```js
// 转译后的样子（大概）
function Hello() {
  return React.createElement('div', null, 'Hello, Claude Code!');
}
```

所以，TypeScript 中的 "React 组件" 在底层就是 **"返回 UI 描述的 JavaScript 函数"**。TypeScript 给它加上了 `Props` 类型约束，但运行时的本质不变。

### Props 就是函数参数

```jsx
// JSX 写法
function Spinner({ mode, loadingStartTimeRef }) {
  return <Text color="green">加载中...</Text>;
}
```

去掉 JSX 和类型后，这就是纯 JavaScript：

```js
function Spinner(props) {
  // props 是一个对象，里面有 mode、loadingStartTimeRef 等属性
  const mode = props.mode;
  const loadingStartTimeRef = props.loadingStartTimeRef;
  return /* 某个 UI 描述 */;
}
```

**对应关系**：
- TS 的 `type Props = { mode: SpinnerMode; ... }` → JS 中就是函数的参数对象 `props`
- TS 的 `Props` 类型检查 → JS 中运行时不检查，传什么就是什么
- TS 的解构写法 `({ mode })` → JS 中就是对象解构 `const { mode } = props`

### `{...props}` 展开运算符传递 Props

```jsx
function Parent(props) {
  return <Child {...props} />;
}
```

转译后就是 JavaScript 的对象展开：

```js
function Parent(props) {
  return Child(props);  // 把 props 对象直接传给 Child
}
```

在纯 JavaScript 中，`{...props}` 就是把父函数收到的整个对象原封不动地传给子函数。

### JSX 不是原生 JavaScript！

**重要**：JSX **不是** JavaScript 的标准语法。浏览器和 Node.js 不能直接运行 JSX，需要**转译器**（如 Babel）把它转换成普通的 JavaScript 函数调用。

### Claude Code 为什么用 JSX？

Claude Code 使用 **Ink** 库在**终端**中渲染 React 组件。也就是说，Claude Code 的界面代码都是 JSX 文件（在源码中是 `.tsx`）。

```jsx
// 来自 src/components/Spinner.tsx（概念性展示）
function Spinner({ mode }) {
  return <Text color="green">加载中...</Text>;
}
```

这里的 `<Text>` 不是 HTML 标签，而是 Ink 提供的终端组件。

### 初学者需要知道的

- JSX 让 UI 代码更直观，看起来像 HTML
- 但它是需要转译的语法扩展，不是纯 JavaScript
- **核心理解**：组件 = 函数，Props = 参数，JSX = 函数返回值的语法糖
- 在 Claude Code 源码中遇到 `.tsx` 文件时，知道里面可能有 JSX 语法即可
- 不需要深入学 JSX，除非你要修改 UI 组件

## 8.10 React Hooks 的 JavaScript 本质

TypeScript 第 8 章讲了很多 React Hooks 的类型（`useState`、`useRef`、`useMemo`、`useEffect`）。这些 Hooks 在底层都是**普通的 JavaScript 函数**。理解它们的 JS 本质，你就不会被 TS 的类型写法吓到。

### useState —— 让函数"记住"值

在 JavaScript 中，普通函数每次调用都会重新执行，局部的变量会被重置：

```js
function clickCounter() {
  let count = 0;
  count = count + 1;
  return count;
}

clickCounter(); // 返回 1
clickCounter(); // 还是返回 1！因为 count 每次都重新初始化为 0
```

React 的 `useState` 用一种特殊的方式让函数"记住"上一次的值。从 JavaScript 的角度看，它类似于：

```js
// 简化理解（不是真实源码）
let stateValue;  // 外部有一个"盒子"保存值

function useState(initialValue) {
  if (stateValue === undefined) {
    stateValue = initialValue;  // 第一次调用，放入初始值
  }
  function setState(newValue) {
    stateValue = newValue;  // 更新盒子里的值
    reRender();             // 触发重新渲染（这里的 reRender 是假设的函数，代表"让界面更新"）
  }
  return [stateValue, setState];  // 返回当前值和更新函数
}
```

**核心理解**：`useState` 返回一个**数组**，第一个是值，第二个是修改值的函数。TypeScript 会标注它们的类型，但运行时的本质就是数组和解构。

```js
const [isOpen, setIsOpen] = useState(false);
// 等价于：
const result = useState(false);
const isOpen = result[0];
const setIsOpen = result[1];
```

### useRef —— 一个不会触发重新渲染的"盒子"

`useRef` 和 `useState` 很像，都是让函数记住值。但区别是：**修改 `useRef` 的值不会触发界面更新**。

```js
// 简化理解
let refValue = { current: 0 };  // 一个对象，值存在 current 属性里

function useRef(initialValue) {
  if (refValue.current === undefined) {
    refValue.current = initialValue;
  }
  return refValue;  // 始终返回同一个对象引用
}
```

**JavaScript 本质**：`useRef` 返回一个**对象** `{ current: ... }`。你修改的是对象的 `current` 属性，而对象本身的引用没变，所以 React 不会认为"状态变了"。

```js
const timerRef = useRef(0);
// timerRef 是 { current: 0 }

timerRef.current = 123;
// 修改了 current 属性，但 timerRef 这个对象引用没变
```

### useMemo —— 缓存计算结果

`useMemo` 的作用是：**只有当依赖发生变化时，才重新计算值**。在 JavaScript 中，这就是"缓存"或"记忆化"的思想：

```js
// 普通写法：每次都会重新计算
function getExpensiveValue(a, b) {
  console.log('重新计算...');
  return a * b * 999999;
}

// Memo 的思想：如果参数没变，直接返回上次的结果
const cache = new Map();

function memoizedCalculate(a, b) {
  const key = `${a},${b}`;
  if (cache.has(key)) {
    return cache.get(key);  // 参数没变，直接拿缓存
  }
  const result = a * b * 999999;
  cache.set(key, result);
  return result;
}
```

**核心理解**：`useMemo(() => compute(), [a, b])` 就是在说："只要 `a` 和 `b` 没变，就别重新计算，用上次的结果。"

### useEffect —— 副作用的注册与清理

"副作用"是指函数除了返回值之外，对外界产生的影响（比如修改全局变量、发网络请求、设置定时器）。

```js
// JavaScript 中的副作用
function setupTimer() {
  const timer = setInterval(() => {
    console.log('每秒执行一次');
  }, 1000);

  // 返回一个"清理函数"
  return function cleanup() {
    clearInterval(timer);
  };
}

// 使用
const cleanup = setupTimer();
// 当不需要时...
cleanup();  // 清理副作用
```

`useEffect` 在 React 中的模式就是：
1. **执行副作用**（设置定时器、订阅事件等）
2. **返回清理函数**（组件卸载或依赖变化时执行）
3. **依赖数组**告诉 React 什么时候重新执行

```js
// 纯 JavaScript 的类比
let previousDeps = null;  // null 表示"还没运行过"
let currentCleanup = null;  // 保存上一次 effect 返回的清理函数

function useEffect(effect, deps) {
  const isFirstRun = previousDeps === null;
  const hasChanged = isFirstRun || deps.some((dep, i) => dep !== previousDeps[i]);

  if (hasChanged) {
    // 先清理上一次的副作用
    if (currentCleanup) currentCleanup();

    // 执行新的副作用
    currentCleanup = effect();
    previousDeps = deps;
  }
}
```

**对应关系**：
- TS 的 `useEffect(() => {...}, [])` → JS 中就是"执行副作用 + 依赖比较 + 清理"
- TS 的依赖数组类型 → JS 中就是一个普通数组，React 逐个比较里面的值

### 自定义 Hooks —— 函数套函数

TypeScript 中的自定义 Hook 如 `useAppState<T>(selector)`，在 JavaScript 中就是**一个接收函数作为参数、返回结果的普通函数**：

```js
// TypeScript 写法
// const isBriefOnly = useAppState(s => s.isBriefOnly);

// JavaScript 本质
function useAppState(selector) {
  const state = getGlobalState();  // 获取全局状态
  return selector(state);          // 把状态传给选择器函数，返回结果
}

// 使用时
const isBriefOnly = useAppState(function(state) {
  return state.isBriefOnly;
});
```

TypeScript 的泛型 `<T>` 只是帮助编译器推断 `selector` 的返回类型。在运行时，JavaScript 只关心"传了一个函数，调用了这个函数，返回了结果"。

### ReactNode —— 函数可以返回任何东西

TypeScript 的 `React.ReactNode` 表示"任何可以渲染的内容"。在 JavaScript 中，这意味着 React 组件的返回值可以是：

```js
function MyComponent() {
  // 以下都是合法的"ReactNode"（在 JavaScript 中就是合法的返回值）
  return 'Hello';           // 字符串
  return 42;                // 数字
  return null;              // null
  return <div>...</div>;    // JSX 元素
  return [1, 2, 3].map(x => <span key={x}>{x}</span>);  // 数组
}
```

TypeScript 用 `ReactNode` 来约束"只能返回这些类型"，但 JavaScript 函数本来就可以返回任何值。

## 8.11 本章小结

### 类与面向对象

| 概念 | 写法 | 作用 |
|------|------|------|
| class | `class Person {...}` | 定义一个类（蓝图） |
| constructor | `constructor(...) {...}` | 构造函数，new 时自动调用 |
| new | `new Person(...)` | 创建类的实例 |
| this | `this.name = name` | 指向当前实例 |
| 方法 | `sayHello() {...}` | 定义实例方法 |
| extends | `class Dog extends Animal` | 继承父类 |
| super() | `super(...)` | 调用父类构造函数 |
| super.method | `super.speak()` | 调用父类方法 |
| getter | `get radius() {...}` | 读取时自动执行 |
| setter | `set radius(v) {...}` | 赋值时自动执行 |
| static | `static add(a,b) {...}` | 静态方法，属于类本身 |
| 私有字段 | `#balance = 0` | 类外部无法访问的属性 |
| instanceof | `obj instanceof Class` | 检查是否为某类的实例 |
| override | JS 中无关键字，TS 用于编译检查 | 标记重写父类方法 |

### JSX 与 React Hooks 的 JS 本质

| 概念 | JS 本质 | 作用 |
|------|--------|------|
| 组件 | 普通函数 | 返回 UI 描述 |
| Props | 函数参数（对象） | 传入数据 |
| `{...props}` | 对象展开运算符 | 传递所有属性 |
| JSX | 语法糖，需转译 | 让 UI 代码更像 HTML |
| useState | 返回 `[值, 设置函数]` 的数组 | 让函数"记住"状态 |
| useRef | 返回 `{ current: ... }` 对象 | 保存值但不触发更新 |
| useMemo | 缓存计算结果 | 避免重复计算 |
| useEffect | 执行副作用 + 返回清理函数 | 处理订阅、定时器等 |
| 自定义 Hook | 普通函数套函数 | 复用状态逻辑 |
| ReactNode | 函数可返回任何值 | 任何可渲染内容 |
